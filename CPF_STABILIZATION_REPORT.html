CPF M3 안정화 리포트

요청서 CPF_NEW_REQUEST 기준으로 ADM 운영 콘솔 핵심 기능 1차 개발을 진행했습니다. 요청 원본 파일은 작업 대상으로 수정하지 않았습니다.

실행하지 않은 검증은 완료로 기록하지 않습니다. 브라우저 실제 클릭, 신규 빈 MariaDB 전체 설치, Redis/Kafka/MQ broker, 표준 헤더 Runtime E2E는 이번 작업에서도 미검증 또는 재확인 필요로 남깁니다.

git 기준 명령재확인 필요샌드박스 사용자와 저장소 소유자가 달라 최초 git 명령은 dubious ownership으로 실패했습니다. 저장소 설정을 바꾸지 않고 명령별 -c safe.directory=...로 기준 정보를 수집했습니다.
사용자 요청 원본미검증CPF_NEW_REQUEST는 읽기 기준으로만 사용했고 기능 검수 대상에서 제외했습니다.
Git commit/push/branch완료수행하지 않았습니다.
별도 변경파일 목록 산출물완료CPF_STABILIZATION_CHANGED_FILES.txt는 생성하지 않았습니다.

구현 요약

기능상태반영 내용
운영자 관리부분 구현기존 AdmOperatorController, AdmOperatorService 구조를 유지하고 runtime smoke 조회 대상에 /adm/api/operators, /adm/api/operators/roles, /adm/api/operators/menus를 포함했습니다. 운영자 쓰기 API runtime smoke는 데이터 오염 위험 때문에 실행하지 않았습니다.
역할 관리부분 구현/adm/api/permissions/roles 목록/상세/등록/수정/상태 변경 API와 DTO, 서비스 SQL 작업, 감사 로그 연계를 추가했습니다.
메뉴 관리부분 구현/adm/api/permissions/menus 목록/상세/등록/수정/상태 변경 API와 상위 메뉴 순환 참조 방지 검증을 추가했습니다.
버튼 권한 관리부분 구현/adm/api/permissions/buttons 목록/상세/등록/수정/상태 변경 API를 추가하고 기존 역할별 버튼 권한 저장 흐름과 감사 로그를 유지했습니다.
API 권한 관리부분 구현adm_api_permission, adm_role_api_permission 테이블과 /adm/api/permissions/api-permissions, /adm/api/permissions/api-matrix, 역할별 API 권한 저장 API를 추가했습니다.
서버 권한검사부분 구현AdmApiAuthFilter가 API 권한 테이블을 먼저 확인하고, 테이블이 없거나 매칭 권한이 없으면 기존 메뉴/버튼 권한으로 fallback합니다. 신규 API 권한 테이블이 적용된 DB에서의 차단/허용 runtime 쓰기 검증은 미검증입니다.
감사 로그부분 구현역할, 메뉴, 버튼, API 권한 생성/수정/상태 변경과 역할별 API 권한 변경에 감사 사유 필수, before/after, diff 메시지를 연결했습니다. 쓰기 API runtime 실행 검증은 미검증입니다.
캐시/메시지/코드/설정부분 구현이번 M3에서는 새 기능 개발 대상이 아니라 runtime smoke 조회 대상에 포함했습니다.
