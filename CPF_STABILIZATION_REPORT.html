CPF 검증 불일치 정리 리포트

이번 작업은 신규 기능 구현이 아니라, 직전 EDU Mapper DB fixture 검증 작업에서 실제 파일 상태와 보고 내용이 어긋난 부분을 정리한 안정화 작업입니다.

요청서는 확인 용도로만 사용했고 수정하지 않았습니다. DB 계정 비밀번호 같은 민감정보는 리포트에 기록하지 않았습니다.

파일 상태

항목결과판단
기존 fixture 경로없음이전 fixture 파일은 현재 저장소에 없습니다. 리포트와 문서는 신규 fixture 경로만 evidence로 사용합니다.
신규 fixture 경로존재xyz/src/test/resources/sql/xyz_edu_query_fixture.sql 파일을 EDU Mapper DB slice 테스트의 기준 fixture로 사용합니다.
변경 파일 목록 산출물없음CPF_STABILIZATION_CHANGED_FILES.txt 파일은 현재 저장소에 없습니다. 앞으로 생기면 HTML 문서 검증이 실패하도록 보강했습니다.
Mapper slice 테스트존재xyz/src/test/java/cpf/xyz/edu/repository/XyzQueryEducationMapperSliceTest.java가 repository 경로에 존재합니다.

수정 내용

대상변경
scripts/check-html-docs.ps1HTML 필수 구조, UTF-8 charset, 첫 non-blank 라인의 Markdown heading, Markdown fence/table/link, specs 하위 Markdown 문서, 루트 Markdown 리포트, 변경 파일 목록 산출물 존재 여부를 검증하도록 강화했습니다.
specs/기능_구현_매트릭스.html상태값을 완료, 부분 구현, 미구현, 미검증, 실패, 재확인 필요 여섯 가지로 제한했습니다. EDU Mapper DB slice는 로컬 검증 사실과 타 환경 재현 필요성을 분리해 재확인 필요로 표기했습니다.
CPF_STABILIZATION_REPORT.html비어 있거나 Markdown처럼 보일 수 있는 내용을 제거하고 실제 HTML 문서로 재작성했습니다. 실행한 검증, 실행하지 않은 검증, 재확인이 필요한 항목을 분리했습니다.

EDU Mapper DB slice 로컬 DB 실행성공별도 환경변수로 로컬 MariaDB 연결을 활성화해 동일 테스트를 재실행했습니다. 민감정보는 리포트에 기록하지 않았습니다.
.\gradlew.bat :xyz:test --offline성공XYZ 모듈 테스트가 성공했습니다.

scripts/check-feature-evidence.ps1성공신규 fixture 경로와 문서 evidence 기준이 통과했습니다.
scripts/check-html-docs.ps1성공HTML 구조와 Markdown 잔재, 불필요 산출물 존재 여부 검사가 통과했습니다.
scripts/check-utf8.ps1 -CheckMojibake성공현재 스크립트 기준 UTF-8 및 mojibake 검사가 통과했습니다.
JUnit XML확인로컬 DB 활성화 후 최종 XyzQueryEducationMapperSliceTest 결과는 tests=3, skipped=0, failures=0, errors=0입니다. DB 환경변수 미설정 기본 경로에서는 skipped=1도 확인했습니다.

판단

항목상태근거와 한계
기존 fixture 제거완료파일 존재 확인 결과 기존 fixture가 없습니다.
변경 파일 목록 산출물 미생성완료파일 존재 확인 결과 산출물이 없습니다.
HTML 문서 구조완료강화한 HTML 문서 검증 스크립트가 통과했습니다.
기능 매트릭스 상태값완료허용 상태값 여섯 가지 기준으로 정리했습니다.
EDU Mapper DB fixture재확인 필요이 PC의 로컬 MariaDB 환경변수 실행은 tests=3, skipped=0로 성공했습니다. 다만 전체 MariaDB 설치 SQL 검증이나 타 PC 재현 완료로 과장하지 않기 위해 기능 매트릭스에는 재확인 필요로 유지했습니다.
MariaDB 전체 신규 설치미검증이번 요청의 제외 범위입니다.
ADM runtime, OpenAPI, 브라우저, broker미검증이번 요청의 제외 범위입니다.
