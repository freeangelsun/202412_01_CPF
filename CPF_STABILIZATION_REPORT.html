CPF 안정화 작업 리포트

작성일: 2026-06-19 KST. 신규 요청 파일은 확인용으로만 사용했고 수정하지 않았습니다. 비밀번호와 secret 값은 리포트에 기록하지 않았습니다.

1. 작업 기준과 재확인

항목결과
요청 기준신규 요청 파일의 “구현하지 않은 것을 완료로 보고하지 말 것”, “요청 파일 수정 금지”, “리포트만 생성” 기준을 적용했습니다.
기존 변경 보존이전 요청 파일 삭제 상태와 신규 요청 파일 untracked 상태는 사용자 변경으로 보고 건드리지 않았습니다.
현행 표준 확인PFW/CMN/ADM/ACC/MBR/BIZADM/EXS/XYZ 모듈 구조, specs/sql, 품질 gate, ADM 정적 UI smoke, MariaDB 설치 SQL을 기준으로 검증했습니다.

2. 구현 완료

영역반영 내용증적
PFW 배치 공통 API Spring Batch 실행을 각 모듈이 직접 호출하지 않도록 cpf.pfw.common.batch에 공통 Facade, 실행 요청/결과 DTO, 이벤트, 잠금, 운영 저장소를 추가했습니다. Batch 의존성은 compileOnly와 조건부 자동구성으로 분리해 비배치 모듈에 BatchAutoConfiguration이 전파되지 않게 했습니다. CpfBatchLauncher, CpfBatchAutoConfiguration, AutoConfiguration.imports
ADM 배치 운영 연동 ADM 배치 실행/재수행/중지 요청은 PFW 공통 CpfBatchLauncher를 사용하도록 정리했고, 스케줄러 잠금은 PFW 공통 CpfBatchLockManager로 통합했습니다. AdmBatchOperationService, PfwBatchScheduler
XYZ 배치 EDU 교육용 배치 실행 컨트롤러가 직접 JobLauncher를 호출하지 않고 PFW 배치 공통 API를 사용하도록 재작성했습니다. Tasklet/Chunk/Retry/Lock/Restart/ADM 연동 안내 API를 유지했습니다. XyzBatchEducationController, XyzEducationController
BIZADM 운영 API 하드코딩 샘플 중심 응답을 운영 DB 기반 조회로 전환했습니다. 기존 /api/bizadm/sample 경로는 호환용 래퍼로 유지하되 운영 서비스에 위임합니다. BizAdmOperationController, BizAdmOperationService, BizAdmOperationRepository
EXS 운영 API 대외기관, 채널, endpoint, 인증 프로파일, token, 라우팅, 거래/전문 로그, 통제 정책, 재처리 API를 DB 기반 운영 기능으로 정리했습니다. 기존 sample 경로도 운영 서비스 위임으로 바꾸고 하드코딩 서비스는 제거했습니다. ExsAdminOperationController, ExsOperationController, ExsOperationService, ExsOperationRepository
SQL/Flyway/문서 동기화 BIZADM 테이블의 _sample suffix를 제거하고 설치 SQL, smoke SQL, Flyway baseline/증분, SQL 가이드와 기능 매트릭스를 같은 이름으로 맞췄습니다. bizadm_menu, bizadm_role, bizadm_permission, bizadm_masking_audit
UTF-8/mojibake 정리 새로 손댄 Java/Gradle/리포트의 깨진 한글을 정상 문장으로 복구했고, check-utf8.ps1의 BOM 및 mojibake 탐지 패턴을 강화했습니다. build.gradle, CmnCoderController, BIZADM/EXS 운영 코드, scripts/check-utf8.ps1

3. MariaDB 실검증

검증상태결과
MariaDB 접속성공로컬 MariaDB 12.3.2 접속 확인. 접속 비밀번호는 기록하지 않음.
00_all_install_and_smoke.sql 실행성공파일 리다이렉션 방식으로 실행 성공. PowerShell 문자열 파이프 방식은 한글 seed가 깨질 수 있어 사용하지 않는 것으로 정리.
거래일자/거래 ID 검증성공transaction_log_date_mismatch_count = 0. 샘플 거래 ID 앞 8자리와 로그일자/시작일자가 일치함.
Spring Batch/CPF 배치 테이블성공BATCH_* JobRepository 테이블과 pfw_batch_* 운영 메타 테이블 생성 확인.
FK/index 확인성공FK: ADM 10, BIZADM 2, CMN 1, MBR 4, PFW 18. Index: ACC 4, ADM 69, BIZADM 56, CMN 28, EXS 45, MBR 34, PFW 143.
seed 재실행성공50_framework_seed_data.sql, 55_cmn_seed_data.sql, 60_adm_seed_data.sql, 70_test_data.sql 재실행 오류 없음.
DB 권한 분리성공cpf_*_app 계정은 DML 권한, cpf_*_migration 계정은 DDL 포함 권한으로 생성 확인. 권한 hash/secret은 기록하지 않음.

4. 검증 결과

명령상태비고
.\gradlew.bat :pfw:compileJava :cmn:compileJava :acc:compileJava :adm:compileJava :xyz:compileJava :bizadm:compileJava :exs:compileJava --offline성공전체 주요 모듈 컴파일 확인.
.\gradlew.bat :bizadm:compileJava :exs:compileJava --offline --rerun-tasks성공BIZADM/EXS 하드코딩 서비스 제거 후 강제 컴파일.
.\gradlew.bat :acc:compileJava --offline --rerun-tasks성공ACC 깨진 주석 정리 후 강제 컴파일.
.\gradlew.bat checkMojibake checkUtf8 --offline성공강화된 mojibake/UTF-8 검사 통과.
.\gradlew.bat qualityGate --offline성공HTML docs, Java format, legacy name, mojibake, security seed, SQL standard, transaction ID standard, UTF-8, compile/test 통과.
.\gradlew.bat admUiSmoke --offline성공ADM 정적 화면/스크립트 smoke 통과.
.\gradlew.bat openApiSmoke --offline실패ADM/XYZ 앱이 기동 중이지 않아 /v3/api-docs 호출 실패. 성공으로 보고하지 않고 앱 기동 후 재검증 항목으로 분리.

5. 일부/미검증/남은 리스크

항목상태사유와 다음 조치
OpenAPI JSON 품질 gate미검증앱 기동이 필요합니다. runLocalServices 또는 ADM/XYZ 단독 기동 후 openApiSmoke를 재실행해야 합니다.
브라우저 실제 클릭 검증미검증정적 ADM UI smoke는 통과했지만, 실제 브라우저 클릭/권한별 비활성 사유/데이터 바인딩은 앱 기동 후 확인해야 합니다.
Redis/Kafka broker 실연동미검증이번 범위에서는 DB fallback과 코드/문서 기준만 확인했습니다. 실제 broker 장애 전환 시나리오는 별도 환경이 필요합니다.
OTP QR/복구코드 실제 앱 호환미검증이번 작업의 직접 변경 범위가 아니며, 실제 OTP 앱 등록/검증 UI/API는 다음 보강에서 기동 검증까지 필요합니다.
요청 파일 인코딩보존신규 요청 파일 자체는 현재 깨진 문자열이 보이지만 요청 파일 수정 금지 조건 때문에 제외했습니다.

6. 다음 보강 후보

  • ADM/XYZ/BIZADM/EXS를 실제 기동한 뒤 OpenAPI JSON 품질 gate와 브라우저 클릭 smoke를 CI에 연결.
