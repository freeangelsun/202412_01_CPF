CPF M2 ADM 브라우저 검증 리포트

이번 작업은 ADM runtime과 OpenAPI가 통과한 상태에서 ADM 운영 콘솔 브라우저 클릭 smoke 가능 여부를 확인하고, 재현 가능한 자동화 진입점을 만드는 작업입니다.

현재 작업 PC에는 Node.js, npm/npx, Playwright, Chrome, Edge, Selenium driver가 없어 실제 브라우저 클릭은 수행하지 못했습니다. 따라서 ADM browser click은 완료가 아니라 미검증으로 기록합니다.

남은 미검증

항목상태사유
ADM browser click미검증Node.js, npm/npx, Playwright, Chrome 또는 Edge가 필요합니다. 준비 후 scripts/smoke-adm-ui.ps1 -BrowserClick -RequireBrowserClick로 재실행합니다.
표준 헤더 Runtime E2E미검증실제 API 호출, DB 거래 로그 저장, ADM 조회, 하위 HTTP 서버 수신까지 묶은 별도 E2E가 필요합니다.
MariaDB 전체 신규 설치미검증기존 로컬 DB runtime smoke는 성공했지만 빈 DB 전체 설치와 seed idempotent 재실행 검증은 별도입니다.
Redis/Kafka/MQ broker미검증실 broker와 장애 fallback 시나리오는 이번 작업에서 실행하지 않았습니다.
운영자/권한/캐시/메시지/코드/설정 API 직접 runtime smoke미검증Controller path는 확인되어 있으나 이번 M2 직접 호출 범위에는 포함하지 않았습니다.
