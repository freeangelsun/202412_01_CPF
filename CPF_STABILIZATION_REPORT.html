CPF 안정화 작업 리포트

요청 파일 CPF_REQUEST.md 기준으로 ADM 운영 기능, 배치 scheduler, 다운로드 감사, 문서, 품질 gate를 보강했습니다.

작업 요약

ADM 알림AdmNotificationController 직접 처리 구조를 AdmNotificationService, DTO, NotificationSender, MockNotificationSender 구조로 분리했습니다. 규칙 조회/상세/등록/수정/비활성화/발송 이력/테스트 발송을 제공합니다.
ADM 다운로드AdmDownloadController, AdmDownloadService, 다운로드 DTO를 추가했습니다. 거래 로그, 오류 로그, 배치 실행 이력, 알림 발송 이력을 CSV로 다운로드하고 사유, 마스킹, 감사 로그를 남깁니다.
DB/SQLadm_download_audit_log를 30_adm_schema.sql, 단일 설치 SQL, baseline Flyway, 신규 V5 migration에 반영했습니다. 알림/다운로드/스케줄러 버튼 권한 seed도 동기화했습니다.
배치 schedulerPfwBatchScheduler, PfwBatchScheduleService, PfwBatchLockService, PfwBatchExecutionTargetService를 추가했습니다. due schedule 판정, 영업일/수행 가능 시간 확인, DB lock, 실행 대상 기록, schedule run-once API를 제공합니다.
ADM 화면ADM 정적 UI를 UTF-8 한글 기준으로 정리하고, 알림/다운로드 메뉴, 다운로드 버튼, 스케줄러 1회 실행 버튼을 연결했습니다.
품질 gatecheckJavaFormat, checkHtmlDocs, openApiSmoke, admUiSmoke를 추가했습니다. qualityGate는 장기 실행 smoke를 제외하고 정적/단위 검증 중심으로 재구성했습니다.
문서루트 README, 문서 인덱스, 관리자/개발자/프레임워크/SQL 가이드, 기능 구현 매트릭스를 이번 변경 기준으로 갱신했습니다. specs 하위 Markdown 문서 잔재는 없습니다.

검증 결과

검증결과비고

.\gradlew.bat test --offline성공기존 단위/슬라이스 테스트 통과
.\gradlew.bat qualityGate --offline성공compile, test, UTF-8, mojibake, legacy name, SQL standard, Java format, HTML docs 통과
powershell -File scripts/check-utf8.ps1 -CheckMojibake성공UTF-8 및 주요 mojibake marker 통과
powershell -File scripts/check-sql-standard.ps1성공테이블 prefix, 공통 컬럼, COMMENT 기준 통과
powershell -File scripts/smoke-adm-ui.ps1성공ADM 정적 메뉴/API marker 검증 통과
MariaDB 00_all_install_and_smoke.sql 실제 실행미검증현재 PATH에서 mysql 클라이언트를 찾지 못해 실행하지 않았습니다.
OpenAPI HTTP smoke미검증ADM/XYZ 앱을 기동하지 않아 /v3/api-docs HTTP 검증은 수행하지 않았습니다.
브라우저 실제 클릭 검증미검증브라우저 자동화 도구가 없어 정적 DOM/API marker smoke까지만 수행했습니다.
scheduler 자동 tick 실시간 검증미검증DB와 앱 장기 기동이 필요한 항목입니다. run-once API와 코드 compile은 검증했습니다.

정리 항목

  • specs/adr 빈 폴더를 제거했습니다.
  • CPF_STABILIZATION_REPORT.md 이전 리포트는 제거하고, 최신 리포트는 이 HTML 파일 하나로 유지했습니다.
  • CPF_STABILIZATION_CHANGED_FILES.txt는 새로 만들거나 갱신하지 않았습니다.
  • 테스트 과정에서 기존 runtime log 산출물 삭제 상태가 감지됐습니다. runtime log는 저장소 관리 대상에서 제외하는 정책이 맞습니다.

남은 리스크

  • MariaDB 실제 smoke는 mysql 클라이언트 경로가 잡힌 뒤 수행해야 합니다.
  • OpenAPI smoke는 ADM/XYZ 앱 기동 후 scripts/smoke-openapi.ps1로 검증해야 합니다.
  • ADM UI는 정적 marker smoke는 통과했지만 실제 브라우저 클릭, 권한별 버튼 비활성/숨김 UX 검증은 추가가 필요합니다.
  • 알림 발송은 mock sender 기준입니다. 실제 Email, Mattermost, Redis/Kafka, Webhook adapter는 운영 채널 결정 후 추가해야 합니다.
  • 배치 scheduler는 기본 비활성입니다. 운영 profile에서 명시적으로 활성화하고 DB lock/장애 시나리오를 검증해야 합니다.
