CPF 안정화 리포트

갱신된 요청 파일을 기준으로 실제 구현, SQL, 문서, 테스트, 로컬 검증을 다시 점검했습니다. 요청 파일은 확인 용도로만 사용했고 작업 대상으로 수정하지 않았습니다.

다음 보강 후보

우선순위보강 후보이유
1앱을 안전하게 기동/종료하는 OpenAPI 검증 자동화현재 smoke 스크립트는 서비스가 떠 있어야 하며, 이번에는 서비스 미기동으로 미검증입니다.
2ADM 브라우저 클릭 자동화정적 smoke는 통과했지만 회원, 배치, 로그, 캐시, 메시지, 코드, 설정 화면의 실제 운영자 UX 검증이 필요합니다.
3ADM 배치 E2E영업일 시뮬레이션, 수행 가능 시간, 선행/트리거 관계, 수행 대상 조회는 존재하지만 실제 scheduler와 Spring Batch 실행 흐름 검증이 필요합니다.
4BIZADM/MBR 인증 API 통합 테스트Service 단위는 보강했지만 Controller, security, DB까지 이어지는 API 통합 테스트가 아직 필요합니다.
5회원 상태 전환과 권한 만료 통합 테스트ADM 회원 관리와 MBR 회원 권한은 운영 기능이므로 등록/수정/잠금/탈퇴/복구/권한 만료 E2E가 필요합니다.
6실 broker 연동과 fallback 장애 시나리오Redis/Kafka/MQ는 로컬 fallback 기준이며, 실 broker 검증은 별도 환경에서 수행해야 합니다.
7EDU 샘플 주석 고도화현재 기능별 EDU 진입점은 있으나 학습용 라인 단위 주석은 더 보강할 수 있습니다.
