이번 작업은 ADM OpenAPI 안전 기동/종료, health check, ADM 배치 API smoke, ADM UI 정적 smoke, 문서 진입점 정리를 중심으로 수행했습니다. 실행하지 않은 검증은 성공으로 표기하지 않았습니다.

수행 작업

영역반영 내용주요 파일
ADM healthADM, PFW, MBR datasource 연결 상태를 확인하는 /adm/api/health API를 추가했습니다. health API는 인증과 거래 헤더 검증에서 제외했습니다.AdmHealthController, AdmApiAuthFilter, TransactionWebMvcConfig
OpenAPI 품질깨진 Swagger annotation을 정상 한글 설명으로 복구하고, OpenAPI smoke tag 기준을 실제 ADM controller tag와 맞췄습니다.AdmAuthController, AdmBatchController, smoke-openapi.ps1
ADM runtime smokeADM boot jar를 기동해 health, OpenAPI, 배치 API, UI marker를 검증하고 스크립트가 시작한 프로세스를 종료하는 자동화 스크립트를 추가했습니다.smoke-adm-runtime.ps1, build.gradle
ADM UI smoke배치 운영 확장 경로인 worker, lock, ghost, operations, relations, execution-targets, calendar marker를 정적 smoke 대상에 추가했습니다.smoke-adm-ui.ps1
문서README와 6개 HTML 가이드를 UTF-8 정상 한글 문서로 재정리하고, PFW와 CMN 역할 경계, ADM runtime smoke 기준, 미검증 기준을 반영했습니다.README, specs HTML 가이드
테스트ADM health 응답의 UP과 DEGRADED 판정을 단위 테스트로 추가했습니다.AdmHealthControllerTest

항상 지켜야 할 기준 점검

기준이번 반영 여부
문서와 소스, SQL, Swagger, EDU 샘플 일치이번 변경 범위인 ADM health, OpenAPI smoke, ADM batch runtime smoke는 소스, 스크립트, README, HTML 가이드, 리포트에 반영했습니다. SQL 변경은 없었습니다.
README는 짧은 진입점, 상세는 가이드 문서README를 진입 문서로 재정리하고 상세 운영/개발/SQL 기준은 HTML 가이드로 분리했습니다.
기능 변경 시 README, 개발가이드, 관리자 가이드, SQL 가이드, EDU 샘플 현행화README와 HTML 가이드는 현행화했습니다. EDU 샘플은 이번 변경 범위에 신규 기능 샘플이 필요하지 않아 추가하지 않았습니다.
신규 주석/설명/SQL COMMENT 한글 작성Java와 HTML 설명은 한글로 작성했습니다. PowerShell 실행 문자열은 Windows PowerShell 인코딩 문제를 피하기 위해 ASCII로 두고, 설명은 문서에 한글로 남겼습니다.
운영 코드 주석은 기능 단위, EDU 주석은 학습용 상세운영 코드에는 health 목적과 smoke 판단 기준을 기능 단위 주석으로 남겼습니다.
최종 리포트에 수행 작업, 검증 결과, 리스크, 보류 항목, 다음 보강 후보 정리이 리포트에 반영했습니다.

남은 리스크와 보류 항목

항목상태사유
브라우저 실제 클릭미검증현재 PATH에 Node, npx, Chrome, Edge, Chromium 계열 실행 파일이 없어 클릭 자동화를 수행하지 못했습니다.
전체 MariaDB 설치 SQL 재실행미실행이번 runtime smoke는 기존 설치 DB를 사용했습니다. 설치 SQL 재실행과 seed idempotent 검증은 별도 작업으로 남깁니다.
배치 장시간 listener heartbeat다음 작업worker 조회와 ghost 조치 API는 smoke했지만, 장시간 실행 Job listener 기반 heartbeat 갱신은 별도 구현 범위입니다.
BAT 독립 실행 모듈다음 작업현재 요청 1순위 범위 밖입니다. 별도 모듈 생성과 JobRepository 운영 시나리오가 필요합니다.
