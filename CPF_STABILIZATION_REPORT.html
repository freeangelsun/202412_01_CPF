CPF 안정화 작업 리포트

작성일시: 2026-06-26 18:45 KST

이번 작업은 로그 정책의 실제 저장 효과와 ADM 거래/오류/감사 관제 기준을 보강하는 범위입니다. 요청서 파일은 확인용으로만 사용했고 작업 대상으로 수정하지 않았습니다. 별도 수정 파일 목록 산출물은 만들지 않았고, 커밋도 수행하지 않았습니다.

수행 작업

영역반영 내용주요 파일
로그 정책 smokerequestBodySaveYn=N, responseBodySaveYn=N, errorStackSaveYn=N이 실제 거래 로그 요약과 상세 로그에서 저장 제외되는지 확인하도록 runtime smoke를 확장했습니다. dbLogEnabled=N 저장 중지와 해제 후 저장 재개도 계속 검증합니다.scripts/smoke-log-policy-runtime.ps1
override와 fallbackactive override 우선 적용, future override 미적용, DB policy fallback, CPF 기본값 fallback을 runtime smoke에 추가했습니다. application 기본값과 expired override fallback은 단위 테스트로 보강했습니다.scripts/smoke-log-policy-runtime.ps1, LogPolicyCacheTest
ADM 로그 관제거래 로그 조회 API에 transactionGlobalId 별칭을 추가해 기존 transactionId와 같은 검색 조건으로 동작하게 했습니다. runtime smoke에서 업무 거래 ID, trace ID, transactionGlobalId 별칭, 실패 로그, 감사 로그 API 도달 상태를 확인합니다.AdmLogController, scripts/smoke-log-policy-runtime.ps1
품질 gate기능 증거 검사에 신규 로그 정책 smoke 항목, ADM transactionGlobalId 별칭, fallback 단위 테스트 증거를 추가했습니다. HTML 문서 검사는 title과 Markdown code fence 잔재까지 확인하도록 강화했습니다.scripts/check-feature-evidence.ps1, scripts/check-html-docs.ps1
문서 현행화README, 문서 인덱스, 관리자 가이드, 운영 매뉴얼, SQL 가이드, 프레임워크 구성 가이드, 기능 구현 매트릭스에 이번 로그 정책 runtime 검증 범위와 남은 리스크를 반영했습니다.README, specs HTML 가이드

Runtime Smoke 결과

항목상태증거
ADM runtime smoke성공health, OpenAPI, 배치 API marker, 거래 메타 scan/list/detail, 정적 UI smoke가 성공했습니다. 브라우저 실제 클릭은 실행 환경 부재로 스크립트 내부에서 skipped로 기록됐습니다.
로그 정책 runtime smoke성공build/runtime-smoke/log-policy-runtime-smoke-result.json 기준 전체 smoke가 완료됐습니다.
dbLogEnabled=N성공override 적용 전후 log index가 1147로 유지되어 DB 로그 저장이 중지됐고, 해제 후 1241로 증가해 저장 재개가 확인됐습니다.
request body 저장 제외성공log index 1246에서 summary REQUEST_BODY와 detail requestBody가 비어 있음을 확인했습니다.
response body 저장 제외성공log index 1246에서 summary RESPONSE와 detail response가 비어 있음을 확인했습니다.
error stack 저장 제외성공의도적 400 오류 거래 log index 1250에서 summary INTERNAL_MESSAGE와 detail error.internalMessage가 비어 있음을 확인했습니다.
fallback 우선순위성공active override는 ADM_OVERRIDE, future override는 시작 전 미적용되어 DB_POLICY, DB 정책이 없는 MODULE 대상은 CPF_DEFAULT로 판정됐습니다. application 기본값과 expired override는 단위 테스트로 확인했습니다.
ADM 로그 조회성공ADM03LGP0014 기준 목록/상세, transactionGlobalId 별칭, trace ID, 실패 로그 조회가 성공했습니다. 감사 로그 API도 HTTP 200으로 응답했습니다.

남은 리스크와 보류 항목

항목상태사유와 다음 조치
브라우저 실제 클릭 자동화미검증현재 runtime smoke 결과에 브라우저 자동화 런타임이 없어 skipped로 기록됐습니다. 브라우저 자동화 환경을 준비한 뒤 ADM 화면 클릭 검증을 별도로 수행해야 합니다.
정책 감사와 일반 감사의 완전 통합부분 구현pfw_log_policy_audit는 정책 도메인 감사 저장소이고, adm_audit_log는 ADM 일반 감사 저장소입니다. 이번에는 두 API/저장소의 상태를 확인했지만 단일 UX 통합은 다음 보강입니다.
Redis/Kafka/MQ 다중 인스턴스 전파보류이번 요청 범위에서 실 broker 연동은 제외됐습니다. 현재는 DB policy와 cache refresh 기준으로 검증했습니다.
운영 부하와 장시간 로그 정책 검증미검증단건 runtime smoke는 통과했지만, 장시간 부하 상태에서 정책 cache TTL, override 만료, 동시 변경 시나리오는 별도 성능/운영 검증이 필요합니다.
전체 설치 SQL 재실행미검증이번 작업은 runtime 정책 검증 중심으로 수행했습니다. 전체 설치 SQL과 seed idempotent 재검증은 별도 DB 초기화 시나리오에서 수행하는 것이 안전합니다.

항상 지켜야 할 기준 점검

기준이번 준수 상태
문서와 소스, SQL, Swagger, EDU 샘플 일치관련 README와 HTML 가이드를 현행화했습니다. 이번 범위는 SQL/EDU 신규 구조 변경이 없어 기존 산출물을 유지했습니다.
README는 짧은 진입점, 상세는 가이드 연결README는 런타임 검증 요약만 보강하고 상세 기준은 specs HTML 가이드에 반영했습니다.
기능 변경 시 문서 동시 현행화관리자 가이드, 운영 매뉴얼, SQL 가이드, 프레임워크 구성 가이드, 기능 구현 매트릭스를 갱신했습니다.
신규 주석과 설명은 한글ADM 로그 컨트롤러 신규 설명은 한글로 작성했습니다. smoke 스크립트의 결과 key는 자동 처리와 JSON 호환성을 위해 영문 식별자를 유지했습니다.
실행하지 않은 검증은 성공으로 보고하지 않음브라우저 클릭, broker 실연동, 전체 설치 SQL 재실행은 미검증 또는 보류로 분리했습니다.

다음 보강 후보

우선순위보강 후보이유
1ADM 거래/오류/정책 감사/일반 감사 통합 UX이번에 API와 smoke 기준은 강화됐지만, 운영자가 한 화면에서 거래와 감사 흐름을 따라가는 UX는 아직 분산돼 있습니다.
2브라우저 클릭 자동화 환경 구성과 ADM 화면 회귀 검증정적 UI marker 검증은 통과하지만 실제 버튼, 탭, 검색, 상세 보기 동작은 별도 자동화가 필요합니다.
3Redis/Kafka/MQ broker 기반 정책 전파 검증다중 인스턴스 운영에서는 cache refresh가 한 인스턴스에만 머물면 안 됩니다.
4로그 정책 부하/장시간/만료 시나리오단건 검증 이후에는 override 만료, TTL, 동시 변경, 대량 트래픽 상태를 검증해야 합니다.
5전체 설치 SQL 초기화 검증새 개발자 배포 기준본으로 굳히려면 초기 DB에서 설치, 재실행, seed idempotent, 권한 분리를 반복 검증해야 합니다.
