CPF 안정화 작업 리포트

요청서의 표준 헤더, DB 표준, 아키텍처 기준, ADM 로그 헤더 분리, EDU 조회 샘플 보강을 중심으로 작업했습니다. 요청 파일은 확인용으로만 사용하고 수정하지 않았습니다.

수행 작업

영역반영 내용상태
PFW 표준 헤더CpfHeaderNames, CpfHeaderSpec, CpfHeaderSpecs, CpfHeaderExtractor, CpfInboundHeaderValidator, CpfHeaderMutator, CpfHeaderPropagator, CpfHeaderMasker, CpfRestClientInterceptor를 추가했습니다.구현
PFW 거래 컨텍스트TransactionHeader를 parent/original transaction, request, correlation, 채널 상세, 운영자, 테넌트, 조직, 위치, 호출자 헤더까지 확장했습니다. TransactionContext에 조회 API와 제한 보정 API 연결점을 추가했습니다.구현
하위 호출 헤더 전파WebClient는 CpfHeaderPropagator.outboundHeaders()를 사용하도록 변경했고, RestClient/RestTemplate 계열은 CpfRestClientInterceptor로 표준 헤더를 적용할 수 있게 했습니다.구현
거래 로그 헤더 스냅샷LoggingAspect가 headers, inboundHeaders, resolvedHeaders, outboundHeaders, responseHeaders를 마스킹 후 detail에 저장하도록 보강했습니다.구현
ADM 로그 상세AdmLogQueryService가 헤더 스냅샷을 별도 응답 영역으로 반환하고, ADM 화면 탭을 수신 헤더, 해석 헤더, 전파 헤더, 응답 헤더로 분리했습니다.구현
EDU 조회 샘플XyzQueryEducationController, XyzQueryEducationService, offset/keyset 응답 DTO를 추가했습니다. 단건, 목록, 검색, 정렬, offset 페이징, keyset 페이징, 헤더 컨텍스트 확인 API를 제공합니다.구현
문서README 진입점을 재정리하고, 표준 헤더 가이드, DB 표준 가이드, 아키텍처 가이드, 트랜잭션 가이드를 추가했습니다. 개발 가이드, 관리자 가이드, 프레임워크 구성 가이드, SQL 가이드, 기능 구현 매트릭스를 연결했습니다.구현
기능 증거check-feature-evidence.ps1에 표준 헤더 구현, 테스트, 신규 문서, EDU 조회 샘플 증거를 추가했습니다. 한글 파일명 비교는 PowerShell 인코딩 영향을 피하도록 ASCII evidence 토큰 기준으로 조정했습니다.구현

온라인 거래 헤더 규격

구분대표 헤더구현 상태
거래 생명주기X-Transaction-Id, X-Parent-Transaction-Id, X-Original-Transaction-Id, X-Request-Id, X-Correlation-Id, X-Idempotency-Key수신 추출, 컨텍스트 저장, 하위 전파, 로그 스냅샷 반영
추적X-Trace-Id, X-Span-Id, X-Parent-Span-Id, traceparent, tracestate현재 span은 응답과 로그에 남기고, 하위 호출에는 parent span으로 전달
채널/주체X-Original-Channel-Code, X-Channel-Code, X-Channel-Detail-Code, X-User-Id, X-Operator-Id, X-Customer-No, X-Member-No필수 헤더 검증, 업무성 헤더 조회와 제한 보정 제공
호출자/위치X-Client-App-Id, X-Client-Version, X-Caller-Service, X-Client-IP, X-Client-Country-Code, X-Client-Timezone수신 추출, 로그 저장, 하위 전파 기준 반영
보안Authorization, X-Api-Key, X-Request-Signature, X-Nonce원문 로그 저장 금지, 업무 코드 보정 차단, 마스킹 기준 반영

MariaDB 설치 SQL 실실행미검증이번 단계에서는 실행하지 않았습니다.
ADM 브라우저 클릭 검증미검증정적 UI 매핑은 반영했지만 브라우저 클릭 smoke는 실행하지 않았습니다.

남은 리스크

  • EDU 조회 샘플은 in-memory 기준입니다. MyBatis Mapper, SQL, 슬라이스 테스트까지 포함한 DB 조회 샘플은 다음 보강이 필요합니다.
  • ADM 헤더 탭은 코드와 응답 구조를 반영했지만 실제 브라우저 클릭 검증은 아직 수행하지 않았습니다.
  • DB 표준 가이드는 작성했지만 대량 DDL 정리는 수행하지 않았습니다. 실제 MariaDB smoke와 migration 설계가 선행되어야 합니다.
  • OpenAPI JSON 품질 gate와 runtime smoke는 이번 검증에서 실행하지 않았습니다.
