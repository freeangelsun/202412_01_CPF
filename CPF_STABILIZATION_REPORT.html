CPF 안정화 작업 리포트

요청서 기준으로 ADM 배치 관제, Spring Batch 운영 메타, worker heartbeat, ghost 조치, lock 운영, SQL/Flyway/문서 동기화를 보강했습니다. 요청서 파일은 확인 용도로만 사용했고 작업 대상으로 수정하지 않았습니다.

민감정보와 DB 비밀번호는 이 리포트에 기록하지 않습니다. 실행하지 못했거나 앱 기동이 필요한 검증은 성공으로 표시하지 않고 미검증으로 분리했습니다.

BAT 모듈 판단

결정사유후속 기준
별도 BAT 모듈 미생성현재 구조는 pfw가 공통 배치 실행기를 제공하고, adm이 관제 API/UI를 제공하며, xyz가 교육 Job을 제공합니다. 별도 모듈을 만들면 중복 런타임과 문서 분기가 커져 이번 범위에는 맞지 않습니다.독립 배치 전용 런타임, 별도 배포 단위, 배치 worker 전용 프로세스가 필요해질 때 BAT 모듈을 재검토합니다.

검증 결과

검증결과비고
MariaDB 접속성공MariaDB 12.3.2 로컬 서비스 접속을 확인했습니다.
전체 설치와 smoke SQL성공00_all_install_and_smoke를 UTF-8 표준입력으로 실행했습니다. worker, ghost, lock, batch execution smoke query가 통과했습니다.
seed idempotent성공cleanup 없이 framework, CMN, ADM, test seed와 smoke query를 재실행했고 row count 중복 증가 없이 통과했습니다.
Flyway V9 SQL성공현재 DB에 V9 SQL을 재적용해 문법 오류 없이 통과했습니다.
SQL 표준성공scripts/check-sql-standard.ps1 통과.
UTF-8/mojibake성공scripts/check-utf8.ps1 -CheckMojibake 통과.
ADM 정적 UI smoke성공scripts/smoke-adm-ui.ps1 통과.
ADM 배치 단위 테스트성공:adm:test --tests cpf.adm.opr.service.AdmBatchOperationServiceTest 통과.
ADM 전체 테스트성공:adm:test --offline 통과.
PFW 테스트 태스크성공:pfw:test --offline는 테스트 소스가 없어 NO-SOURCE였고 컴파일은 통과했습니다.
전체 테스트성공test --offline 통과.
OpenAPI smoke미검증ADM 앱이 기동 중이 아니어서 /v3/api-docs 접속이 실패했습니다. 성공으로 보고하지 않습니다.
브라우저 클릭 검증미검증정적 smoke는 통과했지만 실제 브라우저에서 버튼 클릭과 응답 렌더링은 수행하지 않았습니다.
실 장시간 실행 Job heartbeat미검증이번 구현은 실행 전후 heartbeat 기록과 관제 조회를 보강했습니다. 장시간 chunk 실행 중 주기 heartbeat는 추가 listener 확장 검증이 필요합니다.

남은 리스크

리스크상태다음 조치
OpenAPI 품질 gate앱 기동 필요ADM을 실제 기동한 뒤 smoke 스크립트를 실행하고 CI에서 앱 기동형 gate로 분리합니다.
브라우저 UX 검증미수행ADM 로그인 후 배치 화면의 Job 상세, Worker, Lock, Ghost, 운영 로그 버튼을 실제 클릭 검증합니다.
장시간 실행 중 heartbeat부분 구현Tasklet/Chunk listener 또는 Step listener로 실행 중 주기 heartbeat와 처리율을 갱신합니다.
Ghost 자동 감지수동 조치 구현스케줄러 기반 자동 후보 생성, 알림 연동, 자동 lock release 정책은 별도 정책 확정 후 추가합니다.
배치 관계도 시각화조회 API 구현선행/후행/트리거 관계를 그래프 UI로 표시하는 UX를 추가합니다.
