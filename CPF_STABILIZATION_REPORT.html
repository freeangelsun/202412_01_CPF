CPF 안정화 리포트

이번 리포트는 ADM 거래, 오류, 일반 감사, 정책 감사 통합 관제 기준 보강과 운영 추적 smoke 보강 결과를 기록합니다. 실행하지 않은 검증은 성공으로 표기하지 않습니다.

현재 상태 판정

항목판정근거
리포트 HTML 구조완료실제 HTML 구조로 작성했고 HTML 문서 검증을 통과했습니다.
거래 로그부분 구현/adm/api/logs 목록과 상세 API가 존재하고 거래 ID, trace ID, 업무 거래 ID, 실패 로그 조건 검색을 지원합니다.
오류 로그부분 구현별도 오류 전용 테이블은 없고 pfw_transaction_log의 실패 로그와 상세 로그로 조회합니다. 오류 조치 상태와 이력은 다음 보강입니다.
일반 감사부분 구현/adm/api/audit-logs와 adm_audit_log 기준으로 운영 조치 감사를 조회합니다.
정책 감사부분 구현/adm/api/log-policy-audits를 추가해 pfw_log_policy_audit를 조회합니다.
통합 추적 API부분 구현/adm/api/observability/transactions/{transactionGlobalId}, /adm/api/observability/traces/{traceId}, /adm/api/observability/business-transactions/{businessTransactionId}를 추가했습니다.
ADM 헤더부와 데이터부 표시부분 구현상세 API는 요약, 헤더, 요청, 응답, 오류, 상세, 전문 포맷 데이터를 분리 반환합니다. 브라우저 클릭 검증은 이번 범위에서 제외했습니다.

runtime smokescripts/smoke-log-policy-runtime.ps1에 업무 거래 ID 검색, 통합 추적 API 3종, 일반 감사 API, 정책 감사 API 결과를 admObservability에 기록하도록 보강했습니다.완료

기능 증거 gatescripts/check-feature-evidence.ps1에 신규 API, 서비스, 테스트, smoke marker를 추가했습니다.완료
문서README, 관리자 가이드, 운영 매뉴얼, SQL 가이드, 개발 가이드, 프레임워크 구성 가이드, 기능 구현 매트릭스를 실제 API와 smoke 기준에 맞게 갱신했습니다.완료

온라인 거래 헤더 규격

구분헤더상태설명
필수X-Transaction-Id검증34자리 거래 글로벌 ID입니다. 구조는 17자리 일시, 3자리 모듈 ID, 7자리 WAS ID, 7자리 순번입니다.
필수X-Request-Type검증조회, 등록, 수정, 삭제, 처리 같은 요청 유형입니다.
필수X-Original-Channel-Code검증최초 유입 채널입니다.
필수X-Channel-Code검증현재 처리 채널입니다.
권장X-Trace-Id, X-Parent-Span-Id, X-Span-Id전파trace ID가 없으면 생성하고 span ID는 응답 헤더로 반환합니다. 내부 호출은 parent span을 전달합니다.
권장X-Client-Version, X-Caller-Service, X-Caller-Instance-Id수집클라이언트 버전, 호출 서비스, 인스턴스 정보를 운영 추적에 사용합니다.
권장X-Correlation-Id, X-Idempotency-Key수집외부 연계 상관 ID와 멱등성 키입니다.
권장X-Member-No, X-Customer-No, X-User-Id수집회원, 고객, 사용자 기준 로그 검색에 사용합니다.

TransactionContextFilter는 거래 ID, trace ID, span ID를 응답 헤더로 반환합니다. TransactionHeaderValidationInterceptor는 필수 헤더와 거래 ID 형식을 검사합니다. TransactionContext.propagationHeaders()는 서비스 간 호출에 필요한 표준 헤더를 이어 줍니다.
