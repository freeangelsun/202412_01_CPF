아키텍처 정합성

영역정리 결과
PFW프레임워크 코어와 공통 API 제공자로 정의했습니다. 독립 process로 오해하지 않도록 운영 매뉴얼에도 명시했습니다.
CMN프로젝트 공통 영역으로 정의했습니다. 업무 공통 채번, 업무 알림, 업무 로그는 프로젝트 필요에 따라 확장하는 기준입니다.
BATSpring Batch 기반 실행 worker로 정의했습니다. JobRepository 원천 메타와 CPF 운영 메타 연결 기준을 분리했습니다.
ADM운영 관제 콘솔로 정의했습니다. 실제 Job 실행 책임은 BAT에 두고, ADM은 조회와 조치 요청, 감사 기록을 담당합니다.
center-cutPFW가 업무 target/item/result 테이블을 고정 소유하지 않는 원칙을 문서화했습니다.
