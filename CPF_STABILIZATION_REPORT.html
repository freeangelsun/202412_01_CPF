사전 점검 결과

점검 항목확인 결과조치
BAT 모듈과 health API존재 확인BatHealthController에 heartbeat smoke Job 노출을 유지했습니다.
정상 및 실패 smoke Job존재 확인기존 Job에 CpfBatchRuntimeListener를 연결했습니다.
worker heartbeat와 ghost 테이블존재 확인기존 pfw_batch_worker, pfw_batch_execution, pfw_batch_step_execution, pfw_batch_ghost_event를 확장 사용했습니다.
기존 local DB 상태V10 컬럼 미적용 상태로 확인코드는 구버전 DB에서도 fallback으로 실행되게 유지했고, 신규 설치 SQL과 Flyway V10에 컬럼을 반영했습니다.

구현 내용

영역반영 내용주요 증거
PFW Batch runtimeCPF execution을 Spring Batch 실행 전에 생성하고 JobParameter로 전달하도록 보강했습니다. Job과 Step listener가 같은 execution row에 실행 중 상태와 처리량을 갱신합니다.CpfBatchLauncher, CpfBatchHeartbeatService, CpfBatchRuntimeListener
진행률 메타total, processed, success, failure, retry, progressRate, tps, 평균 및 최대 처리시간, currentStep, lastHeartbeat를 공통 value object로 계산합니다.CpfBatchRuntimeProgress
Step log 보존실행 중 heartbeat log가 Step 종료 시 commit log로 덮이지 않도록 step log를 누적 방식으로 변경했습니다.CpfBatchOperationRepository.upsertStepRuntime
ghost 자동 감지ADM ghost 후보 조회 전에 heartbeat timeout 기준으로 pfw_batch_ghost_event 후보 이벤트를 중복 없이 등록합니다.CpfBatchGhostDetectionService, AdmBatchOperationService.findGhostCandidates
BAT heartbeat smokeCPF_BAT_HEARTBEAT_JOB과 전용 Tasklet을 추가해 최소 2회 이상 heartbeat와 처리 건수 갱신을 검증합니다.BatHeartbeatSmokeTasklet, BatSmokeJobConfig
SQL과 Flyway배치 실행 및 Step 테이블에 진행률과 heartbeat 컬럼을 추가하고 신규 migration을 추가했습니다.V10__batch_heartbeat_progress_ghost_detection.sql, 설치 SQL 합본
문서README, 배치 개발 가이드, 운영 매뉴얼, 관리자 가이드, SQL 가이드, 기능 구현 매트릭스를 현재 코드 기준으로 현행화했습니다.HTML 가이드와 README

검증결과메모
.\gradlew.bat :pfw:compileJava --offline성공PFW 신규 heartbeat와 ghost 서비스 컴파일 확인
.\gradlew.bat :bat:test --offline성공BAT 모듈 테스트 통과
.\gradlew.bat :adm:test --offline성공ADM ghost 자동 감지 호출 테스트 포함
.\gradlew.bat test --offline성공전체 Gradle 테스트 통과
.\gradlew.bat :bat:bootJar --offline성공BAT 실행 jar 생성 확인
scripts/smoke-bat-runtime.ps1성공정상 Job, heartbeat Job, 실패 Job, cleanup 통과. heartbeat Job은 writeCount 4, step log에 heartbeatCount 1부터 4까지 기록됨
scripts/check-sql-standard.ps1성공SQL 표준 검사 통과
scripts/check-utf8.ps1 -CheckMojibake성공UTF-8와 mojibake 검사 통과
scripts/check-feature-evidence.ps1성공기능 증거 검사 통과
.\gradlew.bat qualityGate --offline성공HTML 문서, Java format, legacy name, mojibake, sample, security seed, SQL, transaction ID, 전체 테스트를 포함해 통과
MariaDB V10 migration 실제 적용미검증이번 자동 검증에서는 기존 local DB에 신규 V10을 직접 적용하지 않았습니다. runtime smoke는 구버전 DB fallback 경로로 성공했습니다.

남은 리스크

리스크현재 상태다음 조치
기존 DB의 V10 미적용local smoke 결과에서 확장 컬럼은 아직 비어 있음운영 또는 개발 DB에는 Flyway V10을 적용한 뒤 ADM 목록에서 progress와 lastHeartbeat를 재확인합니다.
ghost 주기 schedulerADM 조회 시 자동 감지까지 구현별도 scheduler로 주기 감지를 수행하고 장애 시나리오 테스트를 추가합니다.
브라우저 클릭 UX 검증미검증ADM 화면 실제 클릭 자동화는 별도 Playwright 또는 브라우저 smoke로 보강합니다.
Redis, Kafka, MQ 장애 시나리오미검증외부 broker 연동과 fallback 장애 시나리오는 별도 환경에서 검증합니다.
