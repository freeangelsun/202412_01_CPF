2. 작업 요약

구분내용
성공BIZADM 인증, MBR 인증, EXS 운영 API를 임시 메모리 저장소 중심에서 DB 저장소 기준으로 보강했습니다.
성공EXS 표준 수신/송신 API를 추가하고 X-Transaction-Id 검증 후 거래/전문 로그를 DB에 선저장하도록 구성했습니다.

일부 구현BIZADM/EXS Vue route와 실제 브라우저 클릭 검증은 아직 수행하지 않았습니다.

7. 정적 확인 항목

  • BIZADM 인증 핵심 구현은 cpf.bizadm.auth로 이동했습니다.
  • EXS 운영 핵심 구현은 cpf.exs.operation, 표준 대외 흐름은 cpf.exs.flow로 이동했습니다.
  • 기존 sample 패키지는 읽기 전용 예제/샘플 성격으로 남겼습니다.
  • 루트 안정화 리포트와 specs HTML 문서는 실제 HTML 구조 검사 대상입니다.
  • 수정파일 목록 산출물과 Markdown 형식 리포트 산출물은 현재 루트에 존재하지 않습니다.

8. 설계 판단 내역

판단근거
DB 비활성 시 503 처리BIZADM/EXS는 DB 없이 가짜 정상 응답을 주지 않도록 datasource 비활성 시 서비스 사용 불가로 처리했습니다.
선택형 datasource교육/문서 확인용 기동과 실제 DB 연동을 분리하기 위해 CPF_BIZADM_DB_ENABLED, CPF_EXS_DB_ENABLED로 활성화합니다.
MBR 로그인 이력 nullable 회원 순번미등록 login ID 실패도 감사 이력으로 남겨야 하므로 member_id를 nullable로 보강했습니다.
EXS 표준 수신 API 거래 ID 검증대외 표준 API에서는 거래 ID를 조용히 생성하지 않고 X-Transaction-Id를 검증해 오류를 명확히 반환합니다.

9. 보안 검증 결과

항목결과
비밀번호 원문 저장저장하지 않음. PBKDF2 hash 기준 검증
refresh token 원문 저장저장하지 않음. hash 저장 후 원문은 응답 1회성 값
EXS 대외 token 원문 저장저장하지 않음. hash와 마스킹 값만 DB 저장
영역 token 혼용 방지BIZADM/MBR loginDomain, issuer, audience 분리
운영 secret 기본값local 기본값은 남아 있습니다. prod profile 강제화는 추가 보강 필요

10. 기능 구현 매트릭스 반영 수준

BIZADM/MBR/EXS 항목은 DB 영속화 반영 상태로 갱신했습니다. 단, 실제 MariaDB HTTP smoke와 Vue 화면 검증은 미검증으로 유지했습니다.

11. 잔여 과제

  1. MariaDB에서 전체 설치 SQL과 smoke SQL 실실행
  2. BIZADM/MBR/EXS DB 활성화 profile로 API smoke 자동화
  3. BIZADM/EXS Vue route, API wrapper, 권한별 버튼 UX 구현
  4. 공통 Bearer token 필터/인터셉터로 보호 API 권한검사 통합
  5. MFA/TOTP QR, 복구코드, challenge 흐름 구현
  6. CMN NotificationSender 공통 adapter를 ADM/BIZADM/MBR/EXS에서 재사용하도록 정리
  7. EDU 예제에 BIZADM/MBR/EXS 인증, EXS 선저장, token refresh, retry 흐름 추가
  8. OpenAPI JSON 품질 gate와 브라우저 UI smoke 자동화
