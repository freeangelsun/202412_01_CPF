CPF EDU Mapper DB Fixture 검증 리포트

신규 요청서 기준으로 XYZ EDU Mapper slice 테스트의 실제 MariaDB fixture 실행 가능성을 검증하고, 성공, skip, 미검증, 실패를 분리했습니다.

신규 요청서는 확인 용도로만 읽었고 수정하지 않았습니다. 별도 수정 파일 목록 산출물은 만들지 않았습니다. DB 인증 정보는 리포트에 기록하지 않습니다.

수정 내용

대상상태내용
xyz/src/test/java/cpf/xyz/edu/repository/XyzQueryEducationMapperSliceTest.java보강정적 XML 안전성 검사, fixture SQL 범위 검사, 실제 DB fixture 실행 검사를 분리했습니다. 신규 환경변수 CPF_XYZ_EDU_MAPPER_DB_USERNAME을 우선 사용하고 기존 CPF_XYZ_EDU_MAPPER_DB_USER는 보조 호환으로 유지했습니다.
xyz/src/test/resources/sql/xyz_edu_query_fixture.sql신규 정리요청 경로에 fixture SQL을 배치했습니다. 반복 실행 가능하도록 CREATE TABLE IF NOT EXISTS, fixture ID 범위 삭제, ON DUPLICATE KEY UPDATE를 사용하고, 전체 삭제와 table drop은 사용하지 않습니다.
scripts/check-feature-evidence.ps1보정EDU Mapper fixture evidence를 새 sql/xyz_edu_query_fixture.sql 경로로 확인하게 했습니다.
specs/개발_가이드.html보정Mapper slice 실DB 검증 환경변수와 fixture 경로를 실제 코드 기준으로 수정했습니다.
CPF_STABILIZATION_REPORT.html신규 작성이번 검증 결과와 남은 미검증 범위를 HTML 리포트로 다시 작성했습니다.

검증 결과

검증결과근거
대상 slice 정적 실행성공.\gradlew.bat :xyz:test --offline --tests cpf.xyz.edu.repository.XyzQueryEducationMapperSliceTest 실행 결과 tests=3, skipped=1, failures=0, errors=0입니다. 안전한 DB 환경변수가 없어서 DB fixture 테스트 1건은 의도대로 skip되었습니다.
대상 slice MariaDB fixture 실행성공로컬 MariaDB의 전용 테스트 스키마를 대상으로 같은 테스트를 --rerun-tasks로 강제 실행했습니다. JUnit XML 기준 tests=3, skipped=0, failures=0, errors=0입니다.
XYZ 전체 테스트성공.\gradlew.bat :xyz:test --offline가 성공했습니다. 일반 실행은 안전 기본값에 따라 DB fixture 환경변수가 없으면 DB 실행부가 skip될 수 있습니다.
전체 Gradle 테스트성공.\gradlew.bat test --offline가 성공했습니다.
품질 게이트성공리포트가 비어 있던 중간 상태에서 checkHtmlDocs가 1회 실패했으나, 리포트 작성 후 .\gradlew.bat qualityGate --offline 재실행이 성공했습니다.
기능 evidence성공scripts/check-feature-evidence.ps1가 새 Mapper slice 테스트와 새 fixture SQL 경로를 확인했습니다.
HTML 문서 검사성공scripts/check-html-docs.ps1가 루트 리포트와 specs HTML 형식을 확인했습니다.
UTF-8 및 mojibake 검사성공scripts/check-utf8.ps1 -CheckMojibake가 통과했습니다.

완료와 미완료 분리

항목상태판단
repository 경로의 Mapper slice 테스트 유지완료xyz/src/test/java/cpf/xyz/edu/repository/XyzQueryEducationMapperSliceTest.java가 실제 존재하고 실행됩니다.
fixture SQL 요청 경로 배치완료xyz/src/test/resources/sql/xyz_edu_query_fixture.sql이 실제 존재하고 테스트에서 로드됩니다.
안전한 MariaDB 테스트 DB fixture 실행완료로컬 전용 테스트 스키마에서 skip 없이 실행됐습니다. 단, 이 검증은 전체 신규 설치 검증이 아니라 Mapper fixture 범위 검증입니다.
MariaDB 전체 신규 설치 검증미검증이번 요청 제외 범위입니다. 전체 설치, FK, index, seed idempotent, 계정 권한 분리는 별도 검증해야 합니다.
ADM runtime, OpenAPI, 브라우저 smoke미검증이번 요청 제외 범위입니다. 앱 기동과 실제 API, 화면 클릭 검증은 수행하지 않았습니다.
표준 헤더 E2E와 broker 검증미검증이번 요청 제외 범위입니다. inbound, context, log, ADM, outbound 흐름과 Redis/Kafka/MQ 실 broker 검증은 남아 있습니다.
