CPF 안정화 리포트

이번 작업은 V13 ADM 권한 seed 실제 적용, PFW 런타임 migration 보정, ADM 거래 메타 E2E smoke 복구, 로그 정책 runtime 효과 검증, 전체 Gradle 테스트 timeout 재확인을 목표로 수행했습니다. 요청서 파일은 작업 대상에서 제외하고 확인 용도로만 사용했습니다.

수행 작업

영역내용결과
PFW DB 보정scripts/apply-pfw-runtime-migrations.ps1를 추가해 V11/V12 런타임 migration을 로컬 MariaDB pfwDB에 적용하고 재실행 idempotent를 확인했습니다.완료
ADM 권한 seedV13 seed에 adm_role_menu 권한 보정을 추가했습니다. 기존 403 원인은 버튼 권한뿐 아니라 메뉴 write 권한이 함께 필요했기 때문입니다.완료
ADM 권한 적용scripts/apply-v13-adm-permission-seed.ps1를 추가해 V13 seed를 실제 로컬 MariaDB admDB에 적용하고 2회 실행 idempotent를 확인했습니다.완료
거래 메타 smokescripts/smoke-adm-runtime.ps1와 scripts/smoke-transaction-meta-runtime.ps1로 ADM 기동, health, OpenAPI, 거래 메타 scan, upsert, 목록, 상세 조회를 확인했습니다.완료
로그 정책 smokescripts/smoke-log-policy-runtime.ps1를 추가해 dbLogEnabled=N override 적용 시 DB 로그 저장이 중지되고, override 해제 후 fallback 저장이 재개되는지 확인했습니다.완료

문서 현행화README, 관리자 가이드, 운영 매뉴얼, SQL 가이드, 프레임워크 구성 가이드, 기능 구현 매트릭스에 신규 적용/검증 명령과 현재 상태를 반영했습니다.완료

미검증 및 보류

항목상태사유
전체 설치 SQL 신규 DB 재설치미실행이번 작업은 기존 로컬 DB에 V11/V12/V13 보강 migration과 권한 seed를 적용하는 범위로 수행했습니다. 전체 drop/create 성격의 설치 검증은 별도 승인된 시점에 실행해야 합니다.
브라우저 실제 클릭 자동화미검증ADM runtime smoke에서 브라우저 자동화 runtime이 PATH에 없어 browser click 항목은 skipped로 기록되었습니다. 정적 UI marker와 API smoke는 통과했습니다.
Redis/Kafka/MQ broker 전파보류이번 요청 범위는 DB fallback과 현재 인스턴스 cache refresh 검증입니다. 실제 broker 장애 시나리오는 별도 환경이 필요합니다.
로그 정책 세부 조합 전체부분 검증이번 smoke는 dbLogEnabled 효과를 실제 로그 순번으로 검증했습니다. request/response/error stack 저장 여부 조합은 후속 케이스로 확대해야 합니다.
