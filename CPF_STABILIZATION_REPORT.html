CPF 안정화 작업 리포트

요청서 파일은 확인 용도로만 사용했고 직접 수정하지 않았습니다. 이번 작업은 표준 헤더 구현 재검수, 로컬 개발환경 문서 추가, EDU 조회 샘플의 MyBatis/SQL 전환, SQL/Flyway 동기화, 가능한 검증 실행을 중심으로 진행했습니다.

실행하지 않은 검증은 성공으로 표시하지 않았습니다. 실제 MariaDB 전체 설치와 ADM runtime smoke는 로컬 DB 초기화와 프로세스 제어 영향이 있어 이번 리포트에서는 미검증으로 분리했습니다.

헤더 규격 상태

구분현재 기준구현 상태
거래 식별X-Transaction-Id, X-Parent-Transaction-Id, X-Original-Transaction-Id, X-Request-Id, X-Correlation-Id, X-Idempotency-Key사전 등록, 필수/권장 분류, outbound 자동 전파 테스트 반영
추적X-Trace-Id, X-Span-Id, X-Parent-Span-Id, traceparent, tracestateSpan은 내부 전용, outbound는 parent span으로 전파
채널과 호출자X-Request-Type, X-Original-Channel-Code, X-Channel-Code, X-Client-Version, X-Caller-Service필수 헤더와 권장 헤더를 사전과 테스트로 고정
사용자 주체X-User-Id, X-Operator-Id, X-Customer-No, X-Member-No회원/고객/운영자 식별을 분리해 로그와 하위 호출에 반영
보안Authorization, X-Api-Key, X-Request-Signature, X-Nonce원문 로그 금지, outbound 전파 금지, 마스킹 정책 테스트 반영
ADM 표시inbound, resolved, outbound, response 헤더 탭정적 코드 기준 구현됨. 브라우저 클릭 기반 확인은 미검증

미검증 항목

항목상태사유와 다음 조치
MariaDB 전체 설치 SQL 실실행미검증00_all_install_and_smoke.sql는 고정 DB명의 테이블 cleanup을 수행합니다. 기존 로컬 DB 데이터를 지울 수 있어 명시적 실행 승인 후 진행해야 합니다.
ADM runtime smoke미검증smoke-adm-runtime.ps1는 앱을 띄우고 포트 소유 프로세스를 정리할 수 있습니다. DB와 포트 사용 상태 확인 후 실행합니다.
로그 정책 runtime smoke미검증실제 ADM 기동과 DB 로그 정책 변경이 필요합니다. 안전한 테스트 DB에서 실행해야 합니다.
OpenAPI runtime smoke미검증ADM/MBR/XYZ/BIZADM/EXS 서비스가 기동된 상태에서만 유효합니다.
ADM 브라우저 클릭 검증미검증정적 UI marker 검증은 통과했지만 실제 브라우저 로그인과 탭 클릭 검증은 수행하지 않았습니다.
