프레임워크 균형 게이트 / V10 실제 DB 적용 / 온라인·배치 공통 운영 기준 정렬

이번 작업은 신규 대형 기능 구현이 아니라 실제 MariaDB V10 적용, BAT heartbeat/progress fallback 제거 검증, 온라인 거래 메타와 로그 정책 상태 판정, 문서와 기능 매트릭스 균형 조정을 목표로 수행했습니다.

현재 상태 판정

항목상태판정 근거
V10 실제 DB 적용완료로컬 MariaDB에 V10 SQL을 직접 적용했고 pfw_batch_execution 11개 컬럼, pfw_batch_step_execution 10개 컬럼, heartbeat index 2개를 information_schema 기준으로 확인했습니다.
BAT heartbeat/progress fallback 제거완료BAT runtime smoke에서 CPF_BAT_HEARTBEAT_JOB이 V10 컬럼 기반 processed_count, progress_rate, last_heartbeat_at을 갱신했습니다.
온라인 거래 메타부분 구현@CpfTransaction과 표준 헤더 검증은 있으나 전용 거래 메타 테이블과 자동 등록 구조는 없습니다.
온라인 로그 정책부분 구현LoggingAspect, TransactionLogListener, pfw_transaction_log 기반 DB 로그 저장은 있습니다. 거래별 파일/DB 로그 정책 테이블과 ADM override는 없습니다.
온라인 오류/감사 로그부분 구현오류는 거래 로그 실패 레코드와 상세 로그로 저장합니다. 별도 pfw_error_log, pfw_audit_log는 없고 운영 감사는 adm_audit_log가 담당합니다.
ADM 온라인 거래 관제부분 구현AdmLogController와 AdmLogQueryService가 거래 로그 검색, JSON pretty, 기본 고정길이 전문 분해를 제공합니다. 거래 메타/로그 정책 관리 API는 없습니다.
배치 메타/로그 정책부분 구현Spring Batch BATCH_*, CPF pfw_batch_*, operation log, ghost 후보 기반은 있습니다. Job/Step별 로그 정책 override는 없습니다.
문서/매트릭스 균형 상태완료README와 HTML 가이드, 기능 구현 매트릭스에 온라인과 배치 상태를 분리해 반영했습니다.

아키텍처 영향

영역영향
PFW프레임워크 코어로 거래 ID, 헤더, 응답, 오류, 로그, Batch 공통 API 기준을 제공한다는 경계를 유지했습니다.
BAT독립 실행 worker와 Spring Batch JobRepository 사용 원칙을 유지하고, V10 진행률 컬럼 기반으로 검증했습니다.
ADM운영 콘솔이 온라인 거래 관제와 배치 관제를 모두 보되 실행 로직을 직접 소유하지 않는 기준을 문서화했습니다.
온라인 거래annotation과 로그 저장은 있으나 메타 자동 등록과 정책 override는 다음 단계로 분리했습니다.
센터컷 확장성이번 범위에서는 구현하지 않고 PFW 기준, BAT 실행, 업무 adapter 확장 원칙만 유지했습니다.
