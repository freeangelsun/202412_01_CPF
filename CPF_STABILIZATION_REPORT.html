CPF 안정화 작업 리포트

이번 작업은 온라인 거래 메타 정본화, 로그 정책 기본 모델, ADM 운영 API와 UI marker, BAT runtime smoke strict 옵션, 문서와 검증 gate 정합성을 보강하는 범위로 진행했습니다. 요청서는 확인용으로만 사용했고 작업 대상으로 수정하지 않았습니다.

수행 작업

영역반영 내용상태
PFW 거래 메타@CpfTransaction 기반 Spring MVC mapping 스캔, pfw_transaction_meta upsert 저장소, 시작 시 자동 스캔 구성을 추가했습니다.부분 구현 완료
ADM 거래 메타/adm/api/transactions 목록, 상세, 재스캔, 비활성화 API와 ADM UI marker를 추가했습니다.부분 구현 완료
로그 정책pfw_log_policy, pfw_log_policy_override, pfw_log_policy_audit 모델과 ADM 관리 API, 감사 사유 검증, override 기간 검증을 추가했습니다.부분 구현 완료
ADM 권한거래 메타와 로그 정책 메뉴, 버튼 권한 seed와 ADM API 권한 매핑을 보강했습니다.부분 구현 완료
BAT smokescripts/smoke-bat-runtime.ps1 기본 경로는 V10 heartbeat/progress 컬럼을 strict 검증하고, 구버전 fallback은 -AllowLegacyFallback 옵션으로 분리했습니다.완료
문서README, 개발 가이드, 관리자 가이드, 프레임워크 구성 가이드, SQL 가이드, 기능 구현 매트릭스에 온라인 거래 메타, 로그 정책, ADM 필수 운영 메뉴, BAT smoke 기준을 반영했습니다.완료

현재 상태 진단

항목판정근거
@CpfTransaction 정본화완료신규 ADM API와 거래 메타 스캐너가 @CpfTransaction을 기준으로 동작합니다. 레거시 이름 검증도 별도 스크립트로 통과했습니다.
거래 메타 자동 등록부분 구현스캐너와 저장소, 시작 시 스캔은 구현했습니다. 실제 ADM 앱 기동 후 운영 DB에 upsert되는 E2E 검증은 별도 운영 검증으로 남습니다.
로그 정책 override부분 구현DB 모델, ADM API, validation, audit 저장은 구현했습니다. 실제 로그 적재 정책에 런타임 적용하는 전파 로직은 다음 단계입니다.
ADM 필수 운영 메뉴부분 구현기능 매트릭스에 필수 운영 메뉴 상태를 명시했고 거래 메타, 로그 정책, 캐시 관리 후보를 반영했습니다.
문서 정합성개선상세 기준은 HTML 가이드에 두고 README는 짧은 진입점으로 유지했습니다.

남은 리스크

리스크설명다음 조치
런타임 정책 적용로그 정책 DB 모델과 ADM API는 있지만 실제 LoggingAspect와 batch listener에 정책 우선순위를 적용하는 로직은 아직 분리되어 있습니다.정책 조회 캐시, override 유효기간 평가, 런타임 전파를 구현합니다.
거래 메타 E2E스캔 구성은 있으나 실제 ADM 앱 기동 후 운영 DB upsert 결과를 자동 검증하지 않았습니다.ADM runtime smoke에 거래 메타 scan과 조회 검증을 추가합니다.
ADM UX거래 메타와 로그 정책 UI marker는 있으나 브라우저 클릭 검증, 상세 오류 메시지, 버튼 비활성 사유 UX는 추가 보강 대상입니다.Playwright 또는 정적 DOM 검증을 도입합니다.
Broker 전파Redis, Kafka, MQ 실 broker 검증은 제외했습니다.DB fallback을 유지한 상태에서 broker별 통합 테스트 profile을 추가합니다.
