CPF 안정화 작업 리포트

이번 작업은 요청서 기준으로 BAT 독립 실행 모듈을 추가하고, Spring Batch JobRepository와 CPF Batch 운영 메타의 연결을 실제 runtime smoke로 확인하는 데 초점을 두었습니다. 실행하지 않은 검증은 성공으로 적지 않고 별도 상태로 남겼습니다.

작업 전 리뷰

항목확인 내용판단
BAT 모듈독립 실행 배치 worker 모듈이 필요했습니다.구현 대상으로 확정
ADM 책임ADM은 실행 주체가 아니라 관제와 조치 콘솔이어야 합니다.책임 분리 유지
SQL 범위요청 범위에서는 신규 center-cut 또는 BAT 전용 테이블을 만들지 않습니다.기존 Spring Batch와 CPF 배치 테이블 사용
검증 기준jar 단독 기동, health, Job/Step E2E, Spring Batch ID와 CPF 메타 ID 연결이 완료 기준입니다.runtime smoke 기준 반영

구분내용다음 조치
브라우저 클릭 검증ADM 실제 화면 클릭 자동화는 이번 범위에서 수행하지 않았습니다.Playwright 또는 Selenium 시나리오를 별도 추가합니다.
운영 배치BAT 독립 실행 기준 Job은 검증했지만, 실제 업무 Job 묶음과 worker pool 배정 정책은 프로젝트별 확장이 필요합니다.업무 Job 템플릿, worker 배정 정책, 장시간 Job listener를 추가합니다.
Broker 연동Redis/Kafka 실 broker 장애 시나리오는 이번 범위가 아닙니다.별도 broker 환경에서 fallback 장애 시나리오를 검증합니다.
전체 DB 재설치전체 설치 SQL 재실행은 데이터 초기화 영향이 있어 이번 작업에서 수행하지 않았습니다.필요 시 별도 승인 후 MariaDB 전체 설치와 seed idempotent를 재검증합니다.
