CPF DB 표준 가이드

이 문서는 DB 소유권, 테이블명, 컬럼명, 공통 감사 컬럼, FK/UK/INDEX 이름, SQL COMMENT, seed, migration, smoke 검증 기준을 정의합니다.

이번 기준은 전수 DDL 변경을 바로 수행하지 않습니다. 대량 테이블 변경은 실DB 영향이 크므로 점검 리포트, migration 설계, MariaDB smoke 검증 후 단계적으로 수행합니다.

DB 소유권

DB소유 영역예시 테이블주의점
pfwDB프레임워크 코어와 운영 메타pfw_transaction_log, pfw_message, pfw_response_code, pfw_config, pfw_batch_*CPF 자체 기능은 PFW에 둡니다.
cmnDB프로젝트 공통 업무cmn_sequence, cmn_notification_log, cmn_business_log프로젝트별 공통 기능 영역이며 프레임워크 코어와 혼동하지 않습니다.
admDB운영자, 권한, 감사adm_operator, adm_role, adm_menu, adm_button, adm_audit_log운영 행위는 감사 사유와 before/after diff를 남깁니다.
mbrDB회원 업무mbr_member, mbr_member_role, mbr_member_login_history회원 테이블은 MBR 소유입니다. ADM은 관리 UI/API만 제공합니다.
bizadmDB, exsDB업무 관리자 예제와 외부 연계 예제bizadm_customer, exs_transaction_log프레임워크 코어 기준과 업무 예제를 분리합니다.

명명 규칙

대상표준예시
테이블{주제영역}_{업무명} lower snake case, _table suffix 금지adm_menu, pfw_message, mbr_member
PKpk_{table}pk_adm_menu
UKuk_{table}_{업무키}uk_pfw_message_code_locale
INDEXidx_{table}_{컬럼요약}idx_pfw_transaction_log_trace
FKfk_{자식테이블}_{부모테이블}_{요약}fk_adm_role_menu_adm_role
컬럼lower snake case를 기본으로 하며 Java/MyBatis 매핑과 일치시킵니다.transaction_id, created_at

공통 컬럼과 COMMENT

컬럼의미기준
created_by등록자모든 업무/운영 테이블에 둡니다.
created_at등록일시기본값은 DB 표준 시각 기준으로 통일합니다.
updated_by수정자수정 API에서 반드시 갱신합니다.
updated_at수정일시수정 시각을 저장합니다.
SQL COMMENT테이블/컬럼 한글 명칭과 설명신규 DDL은 테이블 COMMENT와 주요 컬럼 COMMENT를 반드시 작성합니다.

설치 SQL과 migration

specs/sql/00_all_install.sql과 specs/sql/00_all_install_and_smoke.sql는 단일 실행 파일로 유지합니다. Flyway migration은 변경 이력과 운영 반영 단위를 관리합니다. 두 흐름은 내용이 어긋나지 않게 SQL 가이드와 기능 매트릭스에서 함께 갱신합니다.
