CPF SQL 가이드

이 문서는 CPF DB 소유권, 설치 SQL, Flyway migration, 권한 분리, 배치 메타, smoke 검증 기준을 정리합니다. SQL 변경 시 설치 SQL, migration, seed, smoke, 문서를 함께 맞춥니다.

DB 소유권

DB소유 영역대표 테이블주의사항
pfwDB프레임워크 코어와 운영 메타pfw_transaction_log, pfw_transaction_meta, pfw_log_policy, pfw_message, pfw_response_code, pfw_config, BATCH_*, pfw_batch_*업무 데이터 저장소로 사용하지 않습니다.
cmnDB프로젝트 업무 공통cmn_sequence, cmn_sequence_issue_log, cmn_notification_log, cmn_business_log프로젝트에서 필요 시 optional datasource로 붙입니다.
admDB운영자 권한과 감사adm_operator, adm_role, adm_menu, adm_button, adm_audit_log운영 행위 변경은 감사 로그와 사유를 남깁니다.
mbrDB회원 업무mbr_member, mbr_member_role, mbr_member_login_history, mbr_refresh_token회원 관리는 ADM에서 조회/조치할 수 있으나 소유 DB는 MBR입니다.

exsDB외부 연계 예제exs_institution, exs_channel, exs_endpoint, exs_transaction_log, exs_retry_log외부 연계 토큰은 hash 또는 암호화 기준을 지킵니다.

온라인 거래 메타와 로그 정책 현재 상태

항목테이블/구현현재 판정
온라인 거래 로그pfw_transaction_log, pfw_transaction_log_detail구현되어 있습니다. 요청/응답, 오류, 헤더, 실행 메타를 저장합니다.
온라인 거래 메타 전용 테이블pfw_transaction_metaController/API의 @CpfTransaction, HTTP method/path, Controller class, handler method를 저장합니다.
온라인 오류 로그 전용 테이블pfw_error_log에 해당하는 테이블별도 테이블은 없습니다. 현재는 거래 로그 실패 레코드와 상세 로그로 오류를 추적합니다.
PFW 감사 로그 전용 테이블pfw_audit_log에 해당하는 테이블별도 테이블은 없습니다. 운영 감사는 adm_audit_log가 담당합니다.
로그 정책 테이블pfw_log_policy, pfw_log_policy_override, pfw_log_policy_audit기본 로그 정책, 기간/사유 기반 override, 정책 변경 감사 이력을 저장합니다. 온라인 거래 정책 대상 유형은 ONLINE_TRANSACTION을 표준으로 사용하고 기존 TRANSACTION은 런타임 호환 별칭으로만 처리합니다.
배치 메타BATCH_*, pfw_batch_*구현되어 있습니다. V10 진행률/heartbeat 컬럼은 실제 MariaDB 적용과 smoke 검증까지 완료했습니다.

migration 기준

구분기준
baselineV1__cpf_baseline_install.sql처럼 version prefix를 사용합니다.
신규 변경테이블/컬럼/seed/smoke 변경은 설치 SQL과 migration SQL을 함께 반영합니다.
V10V10__batch_heartbeat_progress_ghost_detection.sql은 pfw_batch_execution과 pfw_batch_step_execution에 진행률, 처리 건수, 마지막 heartbeat 컬럼과 heartbeat 조회 index를 추가합니다.
V11V11__transaction_meta_log_policy.sql은 pfw_transaction_meta와 로그 정책 기본 모델을 추가합니다.
V12V12__log_policy_runtime_standard.sql은 온라인 로그 정책 대상 유형을 ONLINE_TRANSACTION으로 정리하고 기존 override도 같은 표준명으로 갱신합니다.
V13V13__adm_runtime_policy_permission_seed.sql은 기존 ADM DB에 거래 메타 스캔과 로그 정책 캐시 refresh/clear 버튼 및 역할 권한을 비파괴 방식으로 보정합니다.
rollback자동 rollback만 믿지 않고 backup/restore와 수동 복구 절차를 문서화합니다.
idempotent seed재실행해도 중복 데이터가 생기지 않아야 합니다.

V11~V13 로컬 DB 적용 결과

검증결과근거
PFW V11/V12 적용완료scripts/apply-pfw-runtime-migrations.ps1가 pfw_transaction_meta, pfw_log_policy, pfw_log_policy_override, pfw_log_policy_audit 존재와 seed idempotent를 확인합니다.
ADM V13 권한 seed 적용완료scripts/apply-v13-adm-permission-seed.ps1가 TRANSACTION_META, LOG_POLICY 메뉴/버튼 권한과 역할별 재실행 idempotent를 확인합니다.
로그 정책 runtime 효과완료scripts/smoke-log-policy-runtime.ps1가 dbLogEnabled=N 저장 중지와 해제 후 저장 재개, request/response/error stack 저장 제외, active/future override와 DB policy fallback을 확인합니다. application 기본값과 CPF 기본값 fallback은 LogPolicyCacheTest로 보완 검증합니다.

center-cut SQL 기준

PFW는 center-cut 대상 목록, item, result 업무 테이블을 고정으로 소유하지 않습니다. 업무별 데이터 구조와 보존 정책이 다르기 때문입니다. 공통으로 필요한 상태, 오류, 거래 ID, 운영 로그 기준은 PFW/BAT가 제공하고, 실제 업무 target/item/result 테이블은 업무 모듈 또는 BAT 기본 구현체 확장 영역에서 추가합니다.
