CPF SQL 가이드

이 문서는 CPF DB 소유권, 테이블 명명, 설치 SQL, Flyway migration, 권한 계정, smoke 검증 기준을 정리합니다. SQL은 코드와 문서, Swagger, EDU 샘플의 근거가 되므로 가장 먼저 현행화해야 합니다.

과거 테이블명, 대문자 레거시 테이블명, _table suffix는 기준본에 남기지 않습니다. 새로 만드는 단계에서는 정리한 현재 표준만 유지합니다.

테이블 명명 표준

항목기준예시
테이블명{주제영역}_{업무명} lower snake caseadm_menu, pfw_transaction_log, mbr_member
금지 suffix_table 같은 불필요한 suffix 금지pfw_message 사용, message table suffix 금지
공통 컬럼created_by, created_at, updated_by, updated_at모든 업무/운영 테이블 기본 포함
제약조건주제영역 prefix를 포함해 FK, UK, index 이름 통일fk_adm_operator_role_role, uk_mbr_member_login_id
COMMENT테이블과 주요 컬럼 COMMENT는 한글로 작성운영자 ID, 거래 ID, 감사 사유 등

SQL 파일 역할

파일역할주의
specs/sql/01_create_databases.sqlDB 생성로컬 설치 기준이며 운영은 DBA 절차를 따릅니다.
specs/sql/02_create_service_users.sqlmigration 계정과 app 계정 분리app 계정은 DML 중심, DDL은 migration 계정 기준입니다.
specs/sql/10_pfw_schema.sqlPFW 코어와 Spring Batch 테이블프레임워크 메타, 거래 로그, batch 원천/운영 메타를 포함합니다.
specs/sql/20_cmn_schema.sql프로젝트 공통 DB채번, 업무 알림, 업무 로그 등 프로젝트 공통 기능입니다.
specs/sql/30_adm_schema.sqlADM 운영자와 권한운영자, 메뉴, 버튼, 권한, 감사, 다운로드 감사 기준입니다.
specs/sql/40_business_modules_schema.sqlBIZADM, MBR, EXS, ACC 기본 구현체운영 sample이 아니라 DB 기반 업무 모듈 schema입니다.
specs/sql/50_framework_seed_data.sqlPFW seed메시지, 응답코드, 설정, 캐시 이벤트 seed입니다.
specs/sql/55_cmn_seed_data.sqlCMN seed채번 정책, 업무 공통 seed입니다.
specs/sql/60_adm_seed_data.sqlADM seed운영자 역할, 메뉴, 버튼, 권한 seed입니다.
specs/sql/70_test_data.sql로컬 smoke 데이터운영 민감정보 또는 실제 secret을 넣지 않습니다.
specs/sql/99_smoke_check.sql설치 후 확인 queryFK, index, seed, 주요 count를 확인합니다.
specs/sql/00_all_install.sql전체 설치 단일 실행 파일SOURCE 모음이 아니라 실제 SQL 본문을 포함해야 합니다.
specs/sql/00_all_install_and_smoke.sql전체 설치와 smoke 단일 실행 파일실제 MariaDB에서 재실행 idempotent를 확인합니다.

DB 소유권

DB소유 주제대표 테이블운영 기준
pfwDB프레임워크 코어pfw_transaction_log, pfw_response_code, pfw_message, pfw_config, BATCH_*, pfw_batch_*업무 모듈은 PFW API로 접근합니다.
cmnDB프로젝트 업무 공통cmn_sequence, cmn_sequence_issue_log, cmn_notification_log, cmn_business_logoptional datasource로 구성합니다.
admDB운영자 권한과 감사adm_operator, adm_role, adm_audit_log, adm_download_audit_log권한 변경과 운영 조치는 감사 로그를 남깁니다.
bizadmDB업무 관리자 기본 구현bizadm_customer, bizadm_product, bizadm_order, bizadm_masking_audit마스킹 해제 사유와 감사 로그를 필수로 합니다.
mbrDB회원 기본 구현mbr_member, mbr_member_role, mbr_login_history, mbr_refresh_tokenrefresh token 원문 저장 금지입니다.
exsDB대외연계 기본 구현exs_token_store, exs_transaction_log, exs_message_log, exs_retry_log대외 token 원문 저장 금지입니다.

Flyway와 변경 이력

구분기준예시
baseline현재 기준 설치 SQL은 Flyway V1 baseline으로 반영합니다.V1__cpf_baseline_install.sql
변경 migration기능 추가는 V2 이후 순차 migration으로 남깁니다.V8__mbr_auth_login_history_nullable_member.sql
역할 분리설치 SQL은 기준본 설치, Flyway는 운영 변경 이력입니다.둘을 혼용하지 않습니다.
rollback운영 rollback은 별도 복구 SQL 또는 백업 복구 절차로 관리합니다.무분별한 down migration 자동 실행 금지

검증명령 또는 절차성공 기준
