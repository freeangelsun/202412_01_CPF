CPF SQL 가이드

이 문서는 CPF DB 소유권, 설치 SQL, Flyway migration, 권한 분리, seed idempotent, smoke 검증 기준을 정리합니다. DB 표준 세부 기준은 DB 표준 가이드를 함께 봅니다.

DB 소유권

DB소유 영역대표 테이블주의
pfwDB프레임워크 코어와 운영 메타pfw_transaction_log, pfw_transaction_meta, pfw_log_policy, pfw_message, pfw_response_code, pfw_config, BATCH_*, pfw_batch_*업무 데이터 저장소로 사용하지 않습니다.
cmnDB프로젝트 업무 공통cmn_sequence, cmn_sequence_issue_log, cmn_notification_log, cmn_business_log, cmn_edu_query_item프로젝트에서 필요할 때 optional datasource로 붙입니다.
admDB운영자 권한과 감사adm_operator, adm_role, adm_menu, adm_button, adm_api_permission, adm_role_api_permission, adm_audit_log운영 행위 변경은 감사 사유와 before/after diff를 남깁니다.
mbrDB회원 업무mbr_member, mbr_member_role, mbr_member_login_history, mbr_refresh_tokenADM은 운영 관제와 조치 API를 제공하지만 회원 데이터의 소유 DB는 MBR입니다.
bizadmDB업무 관리자 예제bizadm_admin_user, bizadm_customer, bizadm_product, bizadm_order업무 관리자 샘플의 실제 DB 기반 구현체입니다.
exsDB외부 연계 예제exs_institution, exs_channel, exs_endpoint, exs_transaction_log, exs_retry_log토큰과 외부 전문은 원문 노출을 피하고 hash 또는 마스킹 기준을 적용합니다.

설치 SQL

파일역할기준
specs/sql/00_all_install.sql전체 설치 합본SOURCE 지시자가 아니라 실제 SQL 본문을 포함합니다.
specs/sql/00_all_install_and_smoke.sql전체 설치와 smoke query 합본cleanup을 포함하므로 기존 DB에 실행하면 데이터가 삭제될 수 있습니다. 안전한 빈 DB 또는 명시 승인된 DB에서만 실행합니다.
specs/sql/10_pfw_schema.sqlPFW schema프레임워크 코어, Spring Batch, CPF Batch 운영 메타를 포함합니다.
specs/sql/20_cmn_schema.sqlCMN schema프로젝트 공통 기능과 EDU 조회 샘플 테이블을 포함합니다.
specs/sql/30_adm_schema.sqlADM schema운영자, 역할, 메뉴, 버튼, API 권한, 감사, 다운로드 정책을 포함합니다.
specs/sql/40_business_modules_schema.sql업무 예제 schemaACC, MBR, BIZADM, EXS 업무 예제 테이블을 포함합니다.
scripts/build-all-install-sql.ps1합본 SQL 재생성분리 SQL 변경 후 합본과 Flyway baseline을 동기화합니다.

EDU 조회 샘플 SQL

cmn_edu_query_item은 XYZ EDU의 MyBatis 조회 샘플에서 사용하는 교육용 테이블입니다. 회원 번호는 예시 값으로만 보관하고 mbr_member와 직접 조인하지 않습니다. 이 테이블은 20_cmn_schema.sql, 70_test_data.sql, 99_smoke_check.sql, V14__edu_query_sample.sql, 00_all_install.sql, 00_all_install_and_smoke.sql에 같은 기준으로 반영되어야 합니다.

항목기준
단건 조회item_id PK와 use_yn='Y' 조건을 사용합니다.
목록 조회검색어는 item_name, category_code에만 적용합니다.
정렬Mapper XML에서 ID_ASC, NAME_ASC, CREATED_DESC만 허용합니다.
offset 페이징LIMIT, OFFSET, 별도 count query를 사용합니다.
keyset 페이징item_id cursor와 limit + 1 조회로 hasNext를 판단합니다.

검증 명령

powershell -NoProfile -ExecutionPolicy Bypass -File scripts/build-all-install-sql.ps1
mysql --ssl=0 --default-character-set=utf8mb4 -u root -p < specs/sql/00_all_install_and_smoke.sql

실제 MariaDB에서 실행하지 않은 경우 성공으로 기록하지 않고 미검증으로 기록합니다.

check-sql-standard.ps1는 주제영역 prefix, 공통 감사 컬럼, 한글 COMMENT뿐 아니라 split schema의 테이블이 00_all_install.sql, 00_all_install_and_smoke.sql, baseline migration에 누락되지 않았는지도 검사합니다.
