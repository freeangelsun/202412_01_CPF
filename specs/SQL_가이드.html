CPF SQL 가이드

CPF의 설치 SQL, Flyway migration, seed, smoke query, DB 표준을 정리합니다. DB 표준 세부 기준은 DB 표준 가이드에서 함께 확인합니다.

DB 소유권

DB소유 영역주요 테이블주의
pfwDB프레임워크 코어와 운영 메타pfw_transaction_log, pfw_transaction_segment, pfw_log_policy, pfw_message, pfw_response_code, pfw_config, BATCH_*, pfw_batch_*, bat_center_cut_*업무 데이터 저장소로 사용하지 않습니다.
cmnDB프로젝트 업무 공통cmn_sequence, cmn_notification_log, cmn_business_log, cmn_edu_query_item프로젝트에서 필요할 때 optional datasource로 붙입니다.
admDB운영자 권한과 감사adm_operator, adm_role, adm_menu, adm_button, adm_api_permission, adm_audit_log운영 행위 변경은 감사 사유와 before/after diff를 남깁니다.
mbrDB회원 업무mbr_member, mbr_member_role, mbr_member_login_history, mbr_refresh_tokenADM은 운영 API로 관리하지만 회원 데이터 소유권은 MBR에 있습니다.
bizadmDB, exsDB업무 관리자, 외부 연계 예제bizadm_*, exs_*업무 영역 prefix와 공통 감사 컬럼을 유지합니다.

설치 SQL

파일역할기준
specs/sql/00_all_install.sql전체 설치 합본SOURCE 지시자가 아니라 실제 SQL 본문을 포함합니다.
specs/sql/00_all_install_and_smoke.sql전체 설치와 smoke query 합본cleanup을 포함하므로 기존 DB에 실행하면 데이터가 삭제될 수 있습니다.
specs/sql/10_pfw_schema.sqlPFW schema프레임워크 코어, Spring Batch, CPF Batch 운영 메타처럼 PFW가 소유하는 기준 정보를 포함합니다.
specs/sql/35_bat_schema.sqlBAT schemaBAT worker가 소유하는 center-cut 기본 모수, 대상, 결과 테이블을 포함합니다.
specs/sql/50_framework_seed_data.sqlPFW seed코드, 메시지, 응답코드, 설정, batch job, center-cut 모수를 idempotent 방식으로 적재합니다.
scripts/build-all-install-sql.ps1합본 SQL 재생성분리 SQL 변경 후 합본과 Flyway baseline을 동기화합니다.

Reliability persistence 기준

테이블역할운영 조회
pfw_idempotency_recordscope/key/hash, 상태, 저장 응답, TTL과 재시도 허용 여부를 보관합니다.GET /adm/api/reliability/idempotency
pfw_broker_outbox업무 transaction 이후 발행할 메시지, worker lease, attempt/maxAttempts, backoff와 실제 replay 재큐잉 상태를 보관합니다.GET /adm/api/reliability/broker/outbox
pfw_broker_inboxmessageId와 idempotencyKey로 소비 중복을 차단합니다.GET /adm/api/reliability/broker/inbox
pfw_broker_dlq실패 사유, replay 상태·횟수·시각을 보관합니다. replay 요청은 상태만 바꾸지 않고 inbox 중복표시 해제와 outbox PENDING 전환을 함께 수행합니다.GET /adm/api/reliability/broker/dlq
pfw_file_transfer_historytransactionGlobalId, segment, endpoint, checksum, duplicate key와 결과를 보관합니다.GET /adm/api/reliability/file-transfers
pfw_unknown_result외부 결과 미확정 상태와 자동·수동 reconciliation 정보를 보관합니다.GET /adm/api/reliability/unknown-results

V22 source와 all_install에 존재하는 것만으로 실검증 완료가 아닙니다. 신규 빈 MariaDB schema에서 V22 upgrade, PK/UK/index, 재실행 정책을 확인해야 완료로 올립니다.

복합 거래 segment SQL 기준

테이블핵심 컬럼조회 기준
pfw_transaction_segment기본 parent/role 정보와 selected_instance_id, attempt_no, retry_yn, failover_yn, circuit_state, result_state, unknown_result_idADM /adm/api/transaction-groups가 전체 묶음, attempt timeline, header snapshot, external logs를 조회합니다.
indexix_pfw_transaction_segment_global, ..._parent, ..._instance, ..._attempt, ..._unknown거래 tree, 선택 인스턴스, 재시도 순서, unknown reconciliation 검색에 사용합니다.
smokespecs/sql/99_smoke_check.sql설치 후 테이블 존재와 row count를 조회합니다. 실제 runtime 검증은 scripts/smoke-composite-transaction-runtime.ps1와 scripts/smoke-adm-transaction-group-runtime.ps1를 사용합니다.

center-cut SQL 기준

테이블역할확장 기준
bat_center_cut_jobcenter-cut Job 모수PFW/BAT 공통 실행 기준과 ADM 관제 기준을 제공합니다.
bat_center_cut_parametercenter-cut 실행 파라미터secret 값은 직접 저장하지 않고 참조 키 또는 암호화 정책을 사용합니다.
bat_center_cut_item표준 대상 상태업무별 대용량 대상 테이블이 필요한 경우 업무 DB에 별도 target/item/result 테이블을 만들고 adapter로 연결합니다.
bat_center_cut_result표준 결과 이력부모/자식 거래 글로벌 ID와 결과 상태를 운영 추적 기준으로 남깁니다.
