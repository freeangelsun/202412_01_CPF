CPF SQL 가이드

SQL은 주제영역 prefix, 공통 생성/수정 컬럼, FK/UK/index 네이밍, seed idempotent를 기준으로 관리합니다. 설치용 SQL과 migration SQL은 역할을 구분하고, 실행하지 않은 DB 검증은 성공으로 보고하지 않습니다.

DB 소유권

DB소유 영역대표 테이블접근 기준
pfwDB프레임워크 코어pfw_transaction_log, pfw_response_code, pfw_message, pfw_config, BATCH_*, pfw_batch_*업무 모듈은 PFW API로 접근합니다.
cmnDB프로젝트 공통cmn_sequence, cmn_notification_log, cmn_business_log프로젝트 공통 기능이 필요할 때 optional datasource로 붙입니다.
admDB운영자 권한/감사adm_operator, adm_role, adm_menu, adm_button, adm_audit_logADM 운영 기능이 관리합니다.
mbrDB회원 업무mbr_member, mbr_member_role, mbr_login_history회원 업무 모듈이 소유하고 ADM은 운영 조회/조치 API로 접근합니다.
bizadmDB, exsDB업무 관리자와 대외연계bizadm_*, exs_*각 업무 구현체가 소유합니다.

설치 SQL과 Flyway

구분파일역할
분리 설치 SQLspecs/sql/01_create_databases.sql부터 specs/sql/99_smoke_check.sql영역별 설치, seed, smoke를 나누어 관리합니다.
단일 설치 SQLspecs/sql/00_all_install.sqlSOURCE 명령 없이 전체 설치 SQL을 한 번에 실행합니다.
단일 설치와 smokespecs/sql/00_all_install_and_smoke.sql설치 후 smoke query까지 한 번에 실행합니다.
Migrationspecs/sql/migration/flyway/V1__cpf_baseline_install.sql 이후버전 이력과 변경 배포 기준을 남깁니다.

권한 계정 기준

계정권한용도
migration 계정DDL, DML설치와 Flyway migration 실행에 사용합니다.
app 계정DML 중심애플리케이션 런타임에서 사용하며 DDL 권한은 제한합니다.
root로컬 검증 전용개발 PC에서 설치 검증에만 사용하고 운영 문서에는 노출하지 않습니다.
