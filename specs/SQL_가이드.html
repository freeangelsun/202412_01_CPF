CPF SQL 가이드

이 문서는 MariaDB 설치 SQL, Flyway migration, DB 권한, 테이블/컬럼 명명, seed, smoke 검증의 기준서입니다. Java, Mapper, Swagger, ADM 화면과 SQL은 항상 함께 현행화해야 합니다.

신규 설치 기준은 specs/sql/00_all_install.sql과 specs/sql/00_all_install_and_smoke.sql입니다. 이 두 파일은 SOURCE 지시어 목록이 아니라 실제 SQL이 합쳐진 단일 실행 파일이어야 합니다.

DB 소유권

DB책임대표 테이블원칙
pfwDB프레임워크 코어pfw_transaction_log, pfw_message, pfw_response_code, pfw_batch_job, BATCH_*PFW API가 원천 관리합니다.
cmnDB프로젝트 업무 공통cmn_sequence, cmn_sequence_issue_log, cmn_notification_log, cmn_business_log선택형 datasource로 구성하고 미사용 시 서비스가 기동되어야 합니다.

bizadmDB업무 관리자 운영bizadm_admin_user, bizadm_customer, bizadm_download_audit프로젝트 업무 관리자 기본 구현입니다.
mbrDB회원 기본 구현mbr_member, mbr_member_role, mbr_login_history회원 정보와 회원 권한은 ADM 운영자 권한과 분리합니다.
exsDB대외연계exs_institution, exs_transaction_log, exs_message_log, exs_retry_request수신/송신 로그는 업무 처리 전 선저장을 원칙으로 합니다.

Flyway와 migration

항목기준주의
baselineV1__cpf_baseline_install.sql은 신규 설치 기준과 동기화합니다.합본 SQL과 구조가 어긋나면 검증자가 다른 DB를 보게 됩니다.
증분V2__ 이후는 기능 단위로 작성합니다.기존 운영 DB 변경을 위해 idempotent 보정 SQL을 고려합니다.
rollbackDDL rollback 파일보다 보정 migration과 백업/복구 절차를 우선합니다.데이터 손실 가능성이 있는 rollback은 자동화하지 않습니다.
권한migration 계정과 app 계정을 분리합니다.app 계정에 DDL 권한을 주지 않습니다.
