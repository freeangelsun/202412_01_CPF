# ADR 0001. PFW/CMN/ADM/ACC/MBR 주제영역 분리

## 상태

승인

## 결정

- `pfwDB`는 프레임워크 엔진, 메타, 거래로그, 시스템 응답코드, 설정, 보안 메타, 캐시 이벤트를 소유한다.
- `cmnDB`는 업무 공통 기능을 소유한다. 예: 공통 채번, 알림 로그, 공통 업무 로그.
- `admDB`는 운영자, 권한, 메뉴, 세션, 감사 로그, 동적 로그레벨을 소유한다.
- `accDB`, `mbrDB`는 업무 샘플 DB이며 각 업무 prefix 테이블만 가진다.

## 이유

PFW 메타와 업무 공통 데이터를 같은 DB에 섞으면 프레임워크 배포와 업무 확장이 서로 강하게 결합된다. CMN은 jar로 함께 배포하되 `cmnDB` datasource는 선택형으로 두어, 공통 업무 기능이 필요한 프로젝트만 DB를 붙일 수 있게 한다.

## 결과

- DB와 테이블 이름만 봐도 소유 주제영역을 알 수 있다.
- 신규 업무 모듈은 자기 DB 또는 명확한 공통 DB만 참조한다.
- Redis/Kafka 장애 시에도 PFW DB fallback 이벤트로 캐시/로그레벨 동기화를 추적할 수 있다.
