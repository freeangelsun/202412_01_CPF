CPF CoreFlow Platform Framework 문서 인덱스

CPF를 개발, 운영, 검증하는 사람이 어떤 가이드를 먼저 볼지 정리한 진입 문서입니다. README는 개요만 유지하고, 실제 기준은 이 HTML 가이드 묶음에서 관리합니다.

경계 기준: PFW는 프레임워크 코어, CMN은 프로젝트 공통, ADM은 운영 콘솔, BAT는 배치 worker, XYZ는 EDU 샘플입니다. 이 기준은 cpf-pfw-cmn-boundary입니다.

대상먼저 볼 문서다음 확인목적
신규 개발자로컬 개발환경 구성 가이드아키텍처 가이드, 개발 가이드로컬 빌드, DB 설치, 모듈 경계, EDU 샘플을 이해합니다.
온라인 API 개발자표준 헤더 가이드트랜잭션 가이드, 개발 가이드inboundHeaders, 거래 ID, 필수 헤더, CpfRestClientInterceptor 전파 기준을 확인합니다.
DB 담당자SQL 가이드DB 표준 가이드, specs/sql설치 SQL, Flyway, 테이블 prefix, 공통 감사 컬럼, SQL COMMENT 기준을 확인합니다.
운영자관리자 가이드운영 매뉴얼, ADM 화면, ADM API권한, 로그, 배치, 캐시, 메시지, 코드, 설정, 보안 운영 절차를 확인합니다.
배치 개발자배치 개발 가이드bat/src/main/java/cpf/bat, scripts/smoke-bat-runtime.ps1Spring Batch, BAT worker, heartbeat, ghost, center-cut 1차 구조를 확인합니다.
검증자기능 구현 매트릭스최종 안정화 리포트, Gradle 검증 결과구현 상태, 증거 파일, 미검증 항목, 다음 보강 우선순위를 확인합니다.

모듈 책임 요약

모듈책임주요 경로검토 기준
PFW프레임워크 코어pfw/src/main/java/cpf/pfw표준 헤더, 거래 로그, 응답, 예외, OpenAPI, Batch 공통 API가 문서와 일치해야 합니다.
CMN프로젝트 공통cmn/src/main/java/cpf/cmn프로젝트별 공통 기능은 CMN에서 확장하고 PFW 코어를 오염시키지 않습니다.
ADM운영 콘솔adm/src/main/java/cpf/adm, adm/src/main/resources/static/adm/adm/api/transactions, /adm/api/cache, AdmTransactionMetaController, AdmLogPolicyController가 운영 문서와 연결되어야 합니다.
BAT배치 workerbat/src/main/java/cpf/batSpring Batch JobRepository와 CPF 배치 운영 메타가 같은 실행 ID로 연결되어야 합니다.
업무 모듈업무 구현mbr, acc, bizadm, exs업무 DB를 소유하고, PFW 기능은 공통 API로 호출합니다.
XYZ EDU교육 샘플xyz/src/main/java/cpf/xyz/edu학습용 주석, 테스트, 문서 연결을 충분히 제공합니다.

표준 거래 ID

X-Transaction-Id는 온라인 거래 전 구간에서 동일하게 전달되는 34자리 거래 글로벌 ID입니다. 규격은 yyyyMMddHHmmssSSS 17자리, 모듈 ID 3자리, WAS ID 7자리, 일련번호 7자리를 순서대로 조합합니다.

ADM 거래 로그와 상세 로그는 이 값을 기준으로 헤더부, 데이터부, 오류부를 연결합니다. 여러 모듈을 거치는 복합 거래는 pfw_transaction_segment와 /adm/api/transaction-groups 기준으로 구간별 수행시간과 parent 관계를 확인합니다.
