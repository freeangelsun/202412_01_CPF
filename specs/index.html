CPF CoreFlow Platform Framework 문서 인덱스

이 문서는 개발자, 운영자, 검수자가 CPF 프레임워크의 실제 구현과 문서 기준을 같은 순서로 확인하도록 만든 진입점입니다. README는 짧은 개요로 유지하고, 상세 기준은 이 HTML 가이드 묶음에서 관리합니다.

역할 경계는 고정 기준입니다. PFW는 프레임워크 코어 영역이고, CMN은 CPF를 적용하는 프로젝트에서 개발자가 확장하는 프로젝트 공통 영역입니다. PFW 기능은 직접 테이블을 만지는 방식이 아니라 공통 Java API와 운영 API로 사용합니다.

모듈책임주요 경로경계 기준
PFW프레임워크 코어pfw/src/main/java/cpf/pfw거래 ID, 표준 응답, 예외, 로그, OpenAPI, WebClient, Batch 공통 API를 제공합니다.
CMN프로젝트 공통cmn/src/main/java/cpf/cmn업무 채번, 업무 알림, 업무 로그처럼 프로젝트별 공통 기능을 확장합니다.
ADM운영 관리자adm/src/main/java/cpf/adm, adm/src/main/resources/static/admUI 표시, 서버 권한검사, 감사 사유, 마스킹, 다운로드 감사가 함께 동작해야 합니다.
BIZADM업무 관리자 예제bizadm/src/main/java/cpf/bizadm샘플 저장소가 아니라 실제 DB 기반 업무 관리자 기본 구현체입니다.
MBR회원 기본 구현체mbr/src/main/java/cpf/mbr회원 CRUD, 로그인, token hash, 이력, 상태 관리 기준을 제공합니다.
EXS대외연계 기본 구현체exs/src/main/java/cpf/exs기관, 채널, endpoint, token hash, 수신/송신 로그, 재처리 기준을 제공합니다.
XYZ EDU교육 APIxyz/src/main/java/cpf/xyz/edu운영 코드와 구분되는 학습용 코드이며, 주석은 교육 목적상 자세히 작성합니다.

검증 기준

구분명령 또는 기준완료 판단
컴파일.\gradlew.bat compileJava --offline모든 모듈 컴파일 성공
테스트.\gradlew.bat test --offline단위/슬라이스 테스트 성공. 테스트가 없는 기능은 완료로 보지 않습니다.
품질 gate.\gradlew.bat qualityGate --offlineUTF-8, mojibake, SQL 표준, 문서 구조, sample 금지, 기능 증거 검증 통과
ADM 정적 smokescripts/smoke-adm-ui.ps1주요 메뉴, 버튼, API 경로 marker 존재
OpenAPI smokescripts/smoke-openapi.ps1앱 기동 중 JSON 조회 성공. 미기동이면 성공으로 보고하지 않습니다.
ADM runtime smokescripts/smoke-adm-runtime.ps1ADM 기동, health, OpenAPI, 배치 API, 정적 UI smoke, 프로세스 cleanup 수행
MariaDB smokespecs/sql/00_all_install_and_smoke.sql설치, seed 재실행, FK/index, smoke query 확인

거래 추적 기준

X-Transaction-Id는 CPF 온라인 API와 운영 API에서 사용하는 34자리 거래 추적 ID입니다. 구조는 yyyyMMddHHmmssSSS 17자리, module ID 3자리, WAS ID 7자리, sequence 7자리 조합입니다. 호출자가 전달하지 않으면 PFW가 생성하고, 응답 헤더와 거래 로그에 같은 값을 전파합니다.
