CPF CoreFlow Platform Framework 문서 인덱스

이 문서는 CPF를 개발, 운영, 검증하는 사람이 어떤 가이드를 먼저 볼지 정리하는 진입점입니다. README는 짧은 개요만 유지하고, 실제 기준은 이 HTML 문서 묶음에서 관리합니다.

경계 기준: PFW는 프레임워크 코어, CMN은 프로젝트 공통, ADM은 운영 콘솔, BAT는 배치 worker, XYZ는 EDU 샘플입니다.

문서 읽는 순서

대상먼저 볼 문서다음 확인목적
신규 개발자로컬 개발환경 구성 가이드아키텍처 가이드, 개발 가이드, xyz/src/main/java/cpf/xyz/edu로컬 빌드, DB 설치, 모듈 경계, EDU 샘플을 이해합니다.
온라인 API 개발자표준 헤더 가이드트랜잭션 가이드, 개발 가이드거래 ID, 필수 헤더, inboundHeaders, 하위 호출 전파 기준을 확인합니다.
DB 담당자SQL 가이드DB 표준 가이드, specs/sql설치 SQL, Flyway, 테이블 prefix, 공통 감사 컬럼, COMMENT 기준을 확인합니다.
운영자관리자 가이드운영 매뉴얼, ADM 화면, ADM API권한, 로그, 배치, 캐시, 메시지, 코드, 설정, 보안 운영 절차를 확인합니다.
배치 개발자배치 개발 가이드bat/src/main/java/cpf/bat, scripts/smoke-bat-runtime.ps1Spring Batch와 CPF 배치 운영 메타 연동 방식을 확인합니다.
검증자기능 구현 매트릭스최종 안정화 리포트, Gradle 검증 결과구현 상태, 증거 파일, 미검증 항목, 다음 보강 후보를 확인합니다.

모듈 책임 요약

모듈책임주요 경로검수 기준
PFW프레임워크 코어pfw/src/main/java/cpf/pfw표준 헤더, 거래 로그, 예외, 응답, OpenAPI, Batch 공통 API가 문서와 일치해야 합니다.
CMN프로젝트 공통cmn/src/main/java/cpf/cmn프로젝트별 공통 채번, 업무 알림, 업무 로그는 CMN에서 확장하고 PFW 코어와 섞지 않습니다.
ADM운영 콘솔adm/src/main/java/cpf/adm, adm/src/main/resources/static/adm/adm/api/transactions, /adm/api/cache, 로그 상세, 버튼 권한, 감사 사유가 연결되어야 합니다.
BAT배치 workerbat/src/main/java/cpf/batSpring Batch JobRepository와 CPF 배치 운영 메타가 같은 실행 ID로 연결되어야 합니다.
업무 모듈업무 구현mbr, acc, bizadm, exs업무 DB를 소유하고, PFW 기능은 공통 API로 호출합니다.
XYZ EDU교육 샘플xyz/src/main/java/cpf/xyz/edu운영 코드와 분리하고 학습용 한글 주석과 테스트를 충분히 제공합니다.

거래 글로벌 ID 규격

X-Transaction-Id는 CPF 온라인 거래의 34자리 거래 글로벌 ID입니다. 구성은 yyyyMMddHHmmssSSS 17자리, module ID 3자리, WAS ID 7자리, sequence 7자리입니다. 요청에 값이 없거나 형식이 맞지 않으면 PFW가 생성하거나 필수 헤더 검증 단계에서 차단합니다.
