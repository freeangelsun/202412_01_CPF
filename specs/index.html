CPF CoreFlow Platform Framework 문서 포털

CPF는 프레임워크 코어, 프로젝트 공통, 관리자, 회원, 업무 관리자, 대외연계, 교육 API를 함께 제공하는 범용 업무 프레임워크입니다. 이 포털은 개발자, 운영자, 검수자가 같은 기준으로 구현과 문서를 대조하도록 만든 진입점입니다.

PFW는 CPF가 자체 제공하는 프레임워크 코어이고, CMN은 CPF 적용 프로젝트에서 개발자가 확장하는 프로젝트 공통 영역입니다. 이 경계는 SQL, Java API, EDU, 관리자 문서 모두에서 흔들리지 않는 기준입니다.

문서 사용 순서

대상먼저 볼 문서다음 확인목적
신규 개발자개발 가이드프레임워크 구성 가이드, xyz/src/main/java/cpf/xyz/edu신규 업무 API, 배치, 대외연계, 공통 기능 개발 방법을 익힙니다.
운영자관리자 가이드ADM 화면, ADM API, 운영 로그권한, 로그, 배치, 캐시, 메시지, 코드, 설정, 보안 운영 절차를 확인합니다.
DB 담당자SQL 가이드specs/sql, Flyway migrationDB 소유권, 설치 순서, 권한 계정, smoke 검증 기준을 확인합니다.
검수자기능 구현 매트릭스안정화 리포트, qualityGate 결과문서상 완료가 실제 소스, SQL, Swagger, EDU, 검증과 맞는지 대조합니다.

모듈 책임 요약

모듈책임주요 경로주의할 경계
PFW프레임워크 코어와 표준 APIpfw/src/main/java/cpf/pfwPFW DB 원천 기능은 PFW API로 제공하고 업무 모듈이 테이블을 직접 관리하지 않습니다.
CMN프로젝트 공통 확장cmn/src/main/java/cpf/cmn업무 채번, 업무 알림, 업무 로그처럼 프로젝트별 공통은 CMN에서 구현합니다.
ADM프레임워크 운영 관리자adm/src/main/java/cpf/adm, adm/src/main/resources/static/admUI 권한 표시와 서버 API 권한 차단을 함께 맞춥니다.
BIZADM업무 관리자 기본 구현bizadm/src/main/java/cpf/bizadm/operation, bizadm/src/main/java/cpf/bizadm/auth운영 모듈에 sample 패키지를 두지 않습니다.
MBR회원 기본 구현mbr/src/main/java/cpf/mbrADM 회원 관제와 회원 권한 변경 이력을 연결합니다.
EXS대외연계 기본 구현exs/src/main/java/cpf/exs/operation, exs/src/main/java/cpf/exs/flow대외 token 원문은 저장하거나 반환하지 않습니다.
XYZ EDU개발자 교육 APIxyz/src/main/java/cpf/xyz/edu교육 코드라도 공개 클래스명은 Education 용어를 사용합니다.

현재 구현 상태 요약

영역상태근거주의
ADM 운영일부 구현회원, 로그, 권한, 배치, 캐시, 메시지, 코드, 설정, 보안 컨트롤러와 정적 화면 존재실제 브라우저 클릭과 OpenAPI JSON 검증은 앱 기동이 필요합니다.
BIZADM일부 구현DB 기반 운영 컨트롤러와 인증 컨트롤러 존재업무별 확장 기능은 프로젝트에서 추가해야 합니다.
EXS일부 구현대외 기준정보, token, 통제, 재처리, 수신/송신 로그 API 존재외부기관 실연계는 mock/fallback 이후 실환경 검증이 필요합니다.
Batch일부 구현PFW 공통 배치 API, Spring Batch 교육 Job, ADM 배치 관제 API 존재장기 운영 JobRepository 시나리오는 MariaDB와 스케줄러 실기동 검증이 필요합니다.
Docs재작성이 포털과 상세 가이드가 코드 경로와 검증 명령을 연결합니다.기능 추가 시 README, 가이드, SQL, EDU, Swagger를 함께 갱신해야 합니다.
