구현 상태

영역상태
프로젝트명cpf-coreflow-platform-framework
Java packagecpf.* 표준. 기존 Fps* 클래스명은 프레임워크 API 호환 목적으로 유지합니다.
서비스acc(8080), mbr(8081), adm(8090), xyz(8099). pfw, cmn은 라이브러리 모듈입니다.
응답/메시지response_code_table과 message_table을 캐시로 조회하고, 외부/내부 메시지 및 indexed parameter {0} 형식을 지원합니다.
ADM 운영자 권한operator_user, operator_user_role, operator_role, operator_menu, operator_role_menu 기반 DB-first 흐름입니다.
동적 로그레벨현재 WAS 메모리에 즉시 반영하고 dynamic_log_level_rule DB에 저장합니다. 각 WAS는 DB polling으로 룰을 재동기화합니다.
초기 SQLspecs/sql에 생성, seed, 테스트 데이터, smoke check 스크립트를 분리했습니다.
문서, SQL, seed data, 응답코드/메시지, ADM 운영 기능이 바뀌면 관련 문서를 같은 작업에서 함께 갱신합니다.
