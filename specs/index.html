CPF CoreFlow Platform Framework 문서 인덱스

이 인덱스는 개발자, 운영자, 검수자가 같은 기준으로 CPF 프레임워크를 확인하도록 만든 진입점입니다. README는 짧은 개요로 유지하고, 상세 기준은 이 HTML 가이드 묶음에서 관리합니다.

경계 기준: PFW는 프레임워크 코어, CMN은 프로젝트 공통, ADM은 운영 콘솔, BAT는 배치 worker, XYZ는 EDU 샘플입니다.

대상먼저 볼 문서다음 확인목적

모듈 책임 요약

모듈책임주요 경로검수 포인트
PFW프레임워크 코어pfw/src/main/java/cpf/pfw표준 헤더, 거래 로그, 예외, 응답, OpenAPI, Batch 공통 API가 실제 코드와 문서에 일치해야 합니다.
CMN프로젝트 공통cmn/src/main/java/cpf/cmn프로젝트별 채번, 업무 알림, 업무 로그는 CMN에서 확장하고 PFW 코어와 혼동하지 않습니다.
ADM운영 콘솔adm/src/main/java/cpf/adm, adm/src/main/resources/static/adm운영 UI/API는 권한검사, 감사 사유, 마스킹, 로그 상세 포맷을 포함해야 합니다.
BAT배치 workerbat/src/main/java/cpf/batSpring Batch JobRepository와 CPF 배치 운영 메타가 연결되어야 합니다.
업무 모듈업무 구현 예제mbr, acc, bizadm, exs업무 테이블은 각 주제영역 DB가 소유하고 ADM은 운영 관제만 담당합니다.
XYZ EDU교육 샘플xyz/src/main/java/cpf/xyz/edu샘플은 운영 코드와 분리하고 학습용 한글 주석을 충분히 제공합니다.

거래 ID 핵심 규격

X-Transaction-Id는 CPF 온라인 거래의 34자리 거래 글로벌 ID입니다. 구성은 yyyyMMddHHmmssSSS 17자리, module ID 3자리, WAS ID 7자리, sequence 7자리입니다. 요청에 없거나 형식이 맞지 않으면 PFW가 생성하거나 필수 헤더 검증 단계에서 차단합니다.
