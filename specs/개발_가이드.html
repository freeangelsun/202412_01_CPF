CPF 개발 가이드

이 문서는 CPF 기반 신규 업무 개발자가 모듈 경계, 온라인 거래 헤더, Controller/Service/Repository/Mapper 구성, EDU 샘플, 테스트 기준을 확인하는 개발 기준 문서입니다.

핵심 경계는 cpf-pfw-cmn-boundary입니다. PFW는 프레임워크 코어 기능을 제공하고, CMN은 프로젝트 공통 기능을 확장하며, 업무 모듈은 자기 업무 DB와 업무 API를 소유합니다.

신규 업무 모듈 작성 기준

계층작성 대상기준참고 샘플
APIController, Request/Response DTO업무 API는 업무 경로를 사용하고, 운영 API는 /adm/api/{domain} 경로를 사용합니다. 주요 API에는 @CpfTransaction 메타를 둡니다.XyzQueryEducationController, XyzCrudEducationController
Service업무 규칙, 트랜잭션트랜잭션 경계는 Service 계층에 둡니다. 조회는 readOnly=true를 우선 사용합니다.XyzQueryEducationService
Repository조건 정규화, Mapper 호출Controller 파라미터를 SQL에 직접 넘기지 않고 검색어, 상태, 정렬, limit을 정규화합니다.XyzQueryEducationRepository
MapperMyBatis SQL동적 정렬은 whitelist 코드와 XML choose로 처리합니다. 사용자 입력을 ${}로 넣지 않습니다.XyzQueryEducationMapper, XyzQueryEducationMapper.xml
SQLschema, seed, smoke, migration신규 테이블은 주제영역 prefix, 공통 감사 컬럼, 한글 COMMENT, index 이름 기준을 맞춥니다.cmn_edu_query_item, V14__edu_query_sample.sql
Test단위, 슬라이스, 통합 테스트정렬 whitelist, 최대 조회 건수, NotFound, 페이징 hasNext 계산은 단위 테스트로 고정합니다.XyzQueryEducationRepositoryTest, XyzQueryEducationServiceTest

온라인 거래 헤더 사용 기준

헤더 문자열을 직접 만들지 말고 CpfHeaderNames, TransactionContext, CpfHeaderMutator를 사용합니다. 필수 헤더와 하위 호출 전파 기준은 표준 헤더 가이드를 기준으로 합니다.

상황사용 API주의
현재 거래 ID 조회TransactionContext.currentTransactionId()로그, 감사, 외부 호출 상관키로 사용합니다.
현재 회원/고객/채널 조회TransactionContext.memberNo(), customerNo(), channelCode()요청 DTO와 헤더 값이 충돌하면 업무 정책을 명확히 정합니다.
업무 처리 중 헤더 보정CpfHeaderMutator.put(...)거래 ID, trace/span, Authorization, API Key는 업무 코드에서 변경할 수 없습니다.
서비스 간 호출CpfRestClientInterceptor, WebClient filter표준 헤더를 자동 전파하므로 업무 코드에서 반복적으로 header.add를 작성하지 않습니다.
ADM 로그 상세inboundHeaders, resolved, outbound, response 탭민감 헤더는 원문 저장하지 않고 마스킹된 값만 보여줍니다.

EDU 조회 샘플 기준

조회 샘플은 /xyz/edu/query 하위 API에서 확인합니다. 데이터는 cmnDB.cmn_edu_query_item을 사용하고, MBR 같은 업무 테이블과 직접 조인하지 않습니다.

API학습 포인트구현 파일
GET /xyz/edu/query/items/{itemId}단건 조회, NotFound 예외, readOnly 트랜잭션XyzQueryEducationService.getItem
GET /xyz/edu/query/items검색어, 상태, 정렬 whitelist, 최대 limit 제한XyzQueryEducationRepository.findItems
GET /xyz/edu/query/items/pageoffset 페이징, total, hasNextXyzQueryEducationMapper.findOffsetPageItems
GET /xyz/edu/query/items/keyset대용량 목록용 keyset 페이징, limit + 1 방식XyzQueryEducationService.findKeysetPage
GET /xyz/edu/query/headers현재 헤더 컨텍스트와 하위 호출 전파 헤더 확인CpfHeaderPropagator.outboundHeaders
