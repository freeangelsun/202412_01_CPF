개발 가이드

신규 업무 개발자가 CPF 기준으로 Controller, Service, Mapper, SQL, 테스트, 문서를 맞추기 위한 가이드입니다.

API Versioning

신규 외부 공개 API는 /api/v1/{module}/{domain}를 기본으로 사용합니다. 기존 샘플 경로인 /mbr, /accounts, /xyz/edu, /adm/api는 교육/호환 경로로 유지합니다.

신규 모듈 생성 기준

산출물기준
패키지cpf.{module}.{domain}.controller/service/mapper/dto
ControllerSwagger tag, summary, description, request/response example 작성
Service상태 변경 트랜잭션 경계와 권한/감사 사유 검증 위치
Mapper동적 문자열 SQL 조립 금지, parameter binding 사용
SQL{module}_{업무명} 테이블과 공통 감사 컬럼, 한글 COMMENT
Testvalidation, service 단위, mapper 또는 DB smoke

복붙 기준은 신규 업무 모듈 템플릿을 사용합니다.

Batch 개발 샘플

CPF 배치는 Spring Batch를 표준 엔진으로 사용하고, ADM 배치 관제는 Spring Batch 실행 결과와 CPF 운영 메타 테이블을 함께 조회합니다.

Spring Batch 자체 실행 저장소는 BATCH_* 테이블이며, CPF 운영 화면에서 검색하고 감사하는 데이터는 pfw_batch_* 테이블입니다. 신규 배치 개발자는 Job bean을 등록하고, 운영 노출이 필요한 Job은 pfw_batch_job seed 또는 ADM 배치 등록으로 메타를 맞춥니다.

상황참고 샘플운영 연결
단건/정리성 작업POST /xyz/edu/batch/tasklet/runADM 배치 수동 실행과 실행 이력 조회
대용량 chunk 처리POST /xyz/edu/batch/chunk/runstep 단위 read/process/write/skip 건수 확인
실패 재처리POST /xyz/edu/batch/retry/run, GET /xyz/edu/batch/retry-policy실패 이력, 재수행 가능 상태, 감사 사유 연결
중복 실행 방지GET /xyz/edu/batch/lock-policy동일 job/parameter lock과 idempotency 기준
checkpoint/restartGET /xyz/edu/batch/checkpoint-restart재시작 가능 job과 실패 데이터 적재 기준

영업일 수행 정책GET /adm/api/batch/schedules/{scheduleId}/simulation영업일 전용 여부, 휴일 정책, 수행 가능 시간대 사전 확인

수행 대상 인스턴스GET /adm/api/batch/execution-targets수행 대기/배정/완료 대상 인스턴스 조회
