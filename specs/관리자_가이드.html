CPF 관리자 가이드

ADM은 CPF 운영자용 화면과 API입니다. 기본 URL은 http://localhost:8090/adm, API prefix는 /adm/api/**입니다.

1. ADM 개요

항목내용
정적 화면adm/src/main/resources/static/adm/index.html, adm.js, adm.css
인증 필터AdmApiAuthFilter. /adm/api/auth/login을 제외한 /adm/api/** 요청을 검사합니다.
운영자 서비스AdmOperatorService. DB-first, DB 미가용 시 교육용 fallback
세션 서비스AdmSessionService. operator_session token hash 저장

2. 로그인/세션

POST /adm/api/auth/login
{
  "operatorId": "admin",
  "password": "Adm!n12345"
}

응답의 accessToken은 화면에서 Authorization: Bearer {token}으로 사용합니다. 세션은 operator_session에 token hash로 저장되고, 로그아웃 시 폐기합니다.

seed 계정 admin / Adm!n12345은 local/test 전용입니다. 운영 환경에서는 초기 비밀번호 변경, 계정 잠금 정책, secret 외부화를 필수로 적용합니다.

5. 권한

권한은 operator_role_menu를 기준으로 서버에서 메뉴 목록과 READ_YN, WRITE_YN, DELETE_YN을 내려줍니다. UI는 메뉴와 버튼을 필터링하고, AdmApiAuthFilter는 동일 권한으로 API write/delete 요청을 다시 검사합니다.

역할기본 정책
ADM_ADMIN전체 메뉴/작업 가능
ADM_OPERATOR운영 작업 가능, 운영자 관리 제한
ADM_VIEWER조회 중심 권한

6. 감사 로그

주요 운영 조작은 사유를 필수로 받아 operator_audit_log에 저장합니다. 화면의 Audit 메뉴와 GET /adm/api/audit-logs API에서 운영자, action type, target 기준으로 조회합니다.

행위Action Type
운영자 생성OPERATOR_CREATE
운영자 비밀번호 변경OPERATOR_PASSWORD_CHANGE
동적 로그 룰 등록DYNAMIC_LOG_REGISTER
동적 로그 룰 삭제DYNAMIC_LOG_REMOVE
캐시 refreshCACHE_REFRESH
응답코드 등록/수정/삭제RESPONSE_CODE_CREATE, RESPONSE_CODE_UPDATE, RESPONSE_CODE_DELETE

7. 동적 로그레벨

ADM에서 등록한 룰은 즉시 현재 WAS 메모리에 반영하고 dynamic_log_level_rule에도 저장합니다. 각 WAS는 cpf.adm.dynamic-log.sync-millis 주기로 DB 활성 룰을 재동기화합니다.

PUT /adm/api/log-level/rules?businessTransactionId=XYZ01EDU0001&logLevel=DEBUG&ttlSeconds=600&reason=diagnostics
등록/삭제 시 cache_refresh_event에 dynamicLogLevelRule 이벤트를 남기고, CMN 메시징 destination cpf.adm.dynamic-log-level에도 이벤트를 발행합니다. 로컬은 IN_MEMORY, 운영은 Kafka/RabbitMQ 브로커 설정으로 다중 WAS 전파를 구성합니다. Redis Pub/Sub이 필요하면 같은 broadcaster 인터페이스에 Redis adapter를 추가합니다.
