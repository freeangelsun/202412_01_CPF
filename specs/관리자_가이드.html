CPF 관리자 가이드

ADM은 CPF 운영자용 화면과 API입니다. 기본 URL은 http://localhost:8090/adm, API prefix는 /adm/api/**입니다.

1. ADM 개요

항목내용
정적 화면adm/src/main/resources/static/adm/index.html, adm.js, adm.css
인증 필터AdmApiAuthFilter
운영자 서비스AdmOperatorService: DB-first, DB 미가용 시 교육용 fallback
세션 서비스AdmSessionService: operator_session token hash 저장

2. 로그인/세션

POST /adm/api/auth/login
{
  "operatorId": "admin",
  "password": "Adm!n12345"
}

응답의 accessToken은 화면에서 Authorization: Bearer {token}으로 사용합니다. 세션은 operator_session에 token hash로 저장되며 로그아웃 시 폐기됩니다.

4. 권한

권한은 operator_role_menu를 기준으로 서버에서 메뉴 목록을 내려주고, UI는 이 목록으로 표시 메뉴를 필터링합니다.

역할기본 정책
ADM_ADMIN전체 메뉴/작업 가능
ADM_OPERATOR운영 작업 가능, 운영자 관리 제한
ADM_VIEWER조회 중심 권한

5. 감사 로그

주요 운영 조작은 operator_audit_log에 저장합니다.

행위Action Type
운영자 생성OPERATOR_CREATE
운영자 비밀번호 변경OPERATOR_PASSWORD_CHANGE
동적 로그 룰 등록DYNAMIC_LOG_REGISTER
동적 로그 룰 삭제DYNAMIC_LOG_REMOVE

6. 동적 로그레벨

ADM에서 등록한 룰은 즉시 현재 WAS 메모리에 반영되고 dynamic_log_level_rule에도 저장됩니다. 각 WAS는 cpf.adm.dynamic-log.sync-millis 주기로 DB 활성 룰을 재동기화할 수 있습니다.

PUT /adm/api/log-level/rules?businessTransactionId=XYZ01EDU0001&logLevel=DEBUG&ttlSeconds=600&reason=diagnostics
등록/삭제 시 cache_refresh_event에 dynamicLogLevelRule 이벤트도 발행합니다. Redis/Kafka Pub/Sub으로 교체하면 실시간 다중 WAS 전파 구조로 확장할 수 있습니다.

8. 설정

Property설명
cpf.adm.security.enabledADM API 인증 필터 사용 여부
cpf.adm.security.session-ttl-secondsADM 세션 TTL
cpf.adm.password-policy.*비밀번호 길이/복잡도/만료/실패 횟수 정책
cpf.adm.dynamic-log.sync-millisDB 동적 로그 룰 재동기화 주기
운영에서는 DB 비밀번호, JWT signing key, 암호화 secret을 파일에 두지 말고 Vault/KMS 또는 배포 환경변수로 관리합니다.
