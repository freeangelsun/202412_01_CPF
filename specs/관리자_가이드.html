CPF 관리자 가이드

ADM은 프레임워크 운영자가 실제로 조회, 조치, 감사할 수 있어야 하는 운영 콘솔입니다. UI 버튼 표시만으로 완료하지 않고, 서버 권한검사와 감사 사유, before/after diff, 민감정보 마스킹이 함께 적용되어야 합니다.

브라우저 실제 클릭 검증을 수행하지 않은 항목은 화면 검증 완료로 보고하지 않습니다. 정적 smoke는 메뉴와 API marker 확인이며, runtime smoke는 앱 기동과 API 응답 검증입니다.

배치 운영 기준

항목현재 기준추가 고도화
영업일 수행스케줄의 영업일 적용 여부, 수행 가능 시간, 휴일 정책을 기준으로 수행 후보를 계산합니다.달력 월간 뷰와 휴일 일괄 업로드 UX를 확장합니다.
수행 시뮬레이션/adm/api/batch/schedules/{scheduleId}/simulation으로 수행 후보일을 조회합니다.브라우저 클릭 자동화로 결과 표시까지 검증합니다.
선행/후행/트리거 관계pfw_batch_job_relation 기준으로 관계를 조회합니다.그래프형 관계도 UX를 추가합니다.
수행 대상pfw_batch_execution_target 기준으로 대기, 배정, 완료 대상을 조회합니다.대상 재배정과 제외 처리 정책을 확장합니다.
worker heartbeatpfw_batch_worker에서 worker 상태와 heartbeat를 조회합니다.장시간 실행 Job listener 기반 실시간 갱신을 강화합니다.
ghost 조치heartbeat가 끊긴 실행 후보를 조회하고 FAIL, ABANDON, RELEASE_LOCK 조치를 남깁니다.자동 감지 스케줄러와 알림 연계를 고도화합니다.

운영 검증

검증명령결과 해석
정적 UI smokescripts/smoke-adm-ui.ps1ADM 화면과 JS에 핵심 운영 경로가 있는지 확인합니다.
runtime smokescripts/smoke-adm-runtime.ps1앱을 기동하고 health, OpenAPI, 배치 API, UI marker, cleanup을 검증합니다.
브라우저 클릭별도 자동화 런타임 필요런타임이 없으면 성공으로 보고하지 않고 미검증으로 남깁니다.
