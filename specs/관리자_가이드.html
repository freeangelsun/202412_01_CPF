CPF 관리자 가이드

이 문서는 ADM 운영자가 실제 화면과 API를 기준으로 회원, 권한, 로그, 배치, 캐시, 메시지, 코드, 설정, 보안 기능을 어떻게 사용해야 하는지 설명합니다.

ADM 권한은 화면 버튼 표시뿐 아니라 서버 API 권한검사까지 함께 적용해야 합니다. 실행, 재수행, 중지, 잠금해제, 비밀번호 초기화, 다운로드, 마스킹 해제는 감사 사유가 기본입니다.

ADM 진입과 인증

기능화면/API설명운영 주의
로그인/adm, /adm/api/auth/login운영자 ID와 비밀번호로 ADM bearer 세션을 발급합니다.token 원문은 응답 이후 저장하지 않고 hash 기반 세션을 사용합니다.
현재 사용자/adm/api/auth/me운영자와 역할, 허용 메뉴를 조회합니다.메뉴 표시와 서버 권한은 같은 역할 기준이어야 합니다.
로그아웃/adm/api/auth/logout세션을 폐기합니다.강제 폐기는 보안 운영 메뉴에서 별도 감사와 함께 수행합니다.

회원 관리

기능대표 API검색/상세 기준운영 기준
회원 목록/adm/api/members회원번호, 고객번호, 로그인 ID, 이름, 이메일, 휴대폰, 상태, 가입일, 최근 로그인일, 채널개인정보는 기본 마스킹 조회가 원칙입니다.
회원 상세/adm/api/members/{memberId}기본정보, 권한정보, 로그인 이력, 변경 이력, 거래 로그, 감사 로그 연결원문 보기와 다운로드 권한을 별도로 분리합니다.
상태 변경상태 변경, 사용/중지, 잠금/잠금해제, 탈퇴/복구mbr_member, 회원 권한 이력변경 사유와 before/after diff가 필요합니다.
회원 권한역할, 등급, 서비스 접근, 임시 권한, 만료일, 회수mbr_member_role, 권한 이력 테이블ADM 운영자 권한과 회원 권한은 분리하되 ADM에서 관리합니다.

보안 운영

기능운영 절차주의
비밀번호 정책복잡도, 만료, 실패 횟수, 강제 변경 여부를 조회/관리합니다.초기화는 임시 비밀번호 원문을 로그에 남기지 않습니다.
MFA/OTP등록, 검증, 해제, 복구코드 발급을 운영합니다.QR과 secret은 일회성 노출을 원칙으로 합니다.
IP allowlist허용 IP와 사유, 만료일을 관리합니다.운영자 로그인 감사 로그와 함께 확인합니다.
세션 조회/폐기운영자별 세션을 조회하고 강제 폐기합니다.세션 token 원문은 표시하지 않습니다.
민감정보마스킹, 원문 보기 권한, 다운로드 권한을 분리합니다.원문 보기와 다운로드는 감사 사유와 상세 로그를 남깁니다.
