CPF 관리자 가이드

ADM은 프레임워크 운영자가 실제로 조회, 조치, 감사할 수 있어야 하는 운영 콘솔입니다. 운영 기능은 UI 버튼만 있어서는 완료가 아니며, 서버 권한검사와 감사 사유, 마스킹, 이력 저장이 함께 있어야 합니다.

브라우저 실제 클릭 검증을 수행하지 않은 항목은 화면 성공으로 보고하지 않습니다. 정적 smoke는 메뉴와 API marker 확인일 뿐입니다.

ADM 기능 지도

기능화면/APIServiceDB운영 기준
운영자 관리AdmOperatorController, /adm/api/operatorsAdmOperatorServiceadm_operator, adm_operator_role등록, 수정, 잠금, 잠금해제, 비밀번호 초기화는 감사 사유를 요구합니다.
권한 관리AdmPermissionController, /adm/api/permissionsAdmPermissionServiceadm_role, adm_menu, adm_button, adm_api_permission메뉴/버튼/API 권한을 분리하고 서버에서 검사합니다.
회원 관리AdmMemberController, /adm/api/membersAdmMemberOperationServicembr_member, mbr_member_role회원 목록, 상세, 등록, 수정, 상태 변경, 잠금/해제, 권한 변경 이력을 조회합니다.
로그 관제AdmLogController, /adm/api/logsAdmLogQueryServicepfw_transaction_log, pfw_transaction_log_detail, adm_audit_log거래 ID, trace, URI, 응답코드, 사용자, 채널, 성공/실패 조건으로 조회합니다.
배치 관제AdmBatchController, /adm/api/batchAdmBatchOperationService, PfwBatchScheduleServiceBATCH_*, pfw_batch_*목록, 스케줄, 실행 이력, step 로그, 수동 실행, 재수행, 중지, enable/disable을 관리합니다.
캐시 관리AdmCacheControllerAdmCacheOperationServicepfw_cache_refresh_event전체 refresh, key refresh, refresh 이력, 실패 이력을 확인합니다.
메시지/코드/응답코드AdmMessageController, AdmCodeController, AdmResponseCodeController각 운영 Servicepfw_message, pfw_code, pfw_response_code등록/수정/비활성은 감사 사유와 cache refresh를 연결합니다.
설정 관리AdmConfigController설정 운영 Servicepfw_configsecret은 직접 노출하지 않고 마스킹 또는 secret reference만 표시합니다.
다운로드 감사AdmDownloadControllerAdmDownloadServiceadm_download_audit_log다운로드 사유, 건수, 마스킹 여부, 원문 포함 요청 여부, 파일명을 기록합니다.
보안 운영AdmSecurityControllerAdmSecurityOperationService, AdmPasswordPolicyService, AdmSessionServiceADM 보안 테이블비밀번호 정책, OTP, IP allowlist, 세션 폐기, 로그인 감사 로그를 관리합니다.

권한 운영 기준

권한 종류설명서버 검사감사 기준
메뉴 권한화면 접근 가능 여부ADM API 접근 시 역할과 메뉴를 확인합니다.역할별 메뉴 부여/회수는 변경 전/후 diff를 남깁니다.
버튼 권한조회, 등록, 수정, 삭제, 실행, 재수행, 중지, 잠금해제, 비밀번호 초기화 같은 행위 권한버튼을 숨기더라도 API에서 다시 검사합니다.운영 영향이 있는 버튼은 감사 사유를 필수로 합니다.
API 권한서버 endpoint 단위 권한AdmApiAuthFilter와 권한 Service에서 검사합니다.차단 이력은 보안 감사 로그로 남깁니다.
다운로드 권한CSV 다운로드와 원문 포함 여부다운로드 유형과 민감정보 포함 여부를 별도로 검사합니다.사유, 건수, 마스킹 여부, 파일명을 저장합니다.

로그 관제 기준

로그조회 조건상세 표시보안 기준
온라인 거래 로그기간, 거래 ID, trace ID, URI, HTTP status, 응답코드, 회원번호, 고객번호, 채널요약, 헤더, request, response, error, detail민감정보는 기본 마스킹합니다.
감사 로그운영자, 작업 유형, 대상, 사유, 기간before/after diff와 작업 사유삭제 불가 정책을 기준으로 보존합니다.
배치 로그job ID, instance ID, execution ID, 상태, 실행 시간, 실패 사유step log, read/write/skip/retry count재수행/중지는 버튼 권한과 사유가 필요합니다.
전문 로그거래 ID, 외부 거래 ID, 기관, 채널, endpointJSON pretty print 또는 고정길이 필드 표원문 보기와 다운로드 권한을 별도 관리합니다.

배치 운영 기준

항목현재 기준추가 고도화 필요
Job 등록CPF 운영 메타에 job과 스케줄을 등록하고 Spring Batch JobRepository와 연결합니다.등록 UI에서 파라미터 스키마, 선행 job, 트리거 job, 관계도를 더 명확히 보여줘야 합니다.
영업일영업일 테이블과 스케줄 조건을 기준으로 실행 여부를 판단할 수 있는 구조입니다.영업일만 수행, 수행 가능 시간, 다음 수행일 시뮬레이션 버튼은 브라우저 UX 고도화 대상입니다.
중복 실행 방지동일 job/parameter 기준 lock과 상태 조건을 확인합니다.실제 동시 실행 부하 시나리오 검증이 필요합니다.
재수행실패 상태와 사유를 기준으로 재수행 요청을 기록합니다.checkpoint/restart 대상 step 표시와 실패 데이터 재처리 UI가 필요합니다.
