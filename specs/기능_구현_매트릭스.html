CPF 기능 구현 매트릭스

이 문서는 CPF 기능별 구현 상태와 검증 상태를 분리해서 표시하는 검수 기준 문서입니다. 구현 파일이 있어도 실행하지 않은 runtime, 브라우저, MariaDB, broker 검증은 성공으로 쓰지 않습니다.

완료는 소스, SQL, 문서, 테스트 또는 smoke 검증 증거가 현재 저장소 기준으로 모두 확인된 경우에만 사용합니다. 정적 marker 검증은 runtime 검증이나 브라우저 클릭 검증으로 간주하지 않습니다.

상태값 기준

상태정의
완료소스, SQL, 문서, 테스트 또는 smoke 증거가 있고 이번 작업에서 실행 결과를 확인했습니다.
부분 구현핵심 구조는 있으나 runtime, 브라우저, 실DB, 외부 연동, E2E 중 일부가 남았습니다.
미구현구현 파일이나 동작 경로가 없습니다.
미검증구현 또는 스크립트는 있으나 이번 작업에서 실행하지 않았습니다.
실환경 필요실제 MariaDB, Redis/Kafka/MQ, 브라우저 자동화, 외부기관 등 별도 환경이 필요합니다.
재확인 필요로컬 상태, 브랜치 차이, 설정 차이로 추가 확인이 필요합니다.

핵심 기능 상태

영역기능상태증거남은 확인
PFW표준 헤더 사전과 필수 헤더 검증부분 구현CpfHeaderNames, CpfHeaderSpecs, CpfInboundHeaderValidator, TransactionHeaderValidationInterceptor, PFW 단위 테스트ADM runtime/OpenAPI에서 실제 요청 기준 확인은 미검증입니다.
PFWtrusted proxy 기반 client IP 해석부분 구현CpfTrustedProxyPolicy, CpfHeaderExtractor, LoggingAspect, CpfHeaderStandardCoverageTestExtractor 실제 연결, CIDR, X-Forwarded-For, Forwarded, invalid IPv4/hostname/unknown/blank 값 무시, bracket IPv6 정규화는 단위 테스트로 확인합니다. 실제 LB/WAF 뒤의 운영 트래픽 검증은 실환경 필요입니다.
PFW업무 코드 헤더 보정 제한부분 구현CpfHeaderMutator, CpfHeaderMutatorTest, 표준_헤더_가이드.htmlX-User-Id, X-Operator-Id 차단, 대소문자 우회 차단, 실패 시 context 보존은 단위 테스트로 확인합니다. 운영 예외 API와 감사 diff 흐름은 별도 기능입니다.
PFWWebClient/RestClient 하위 호출 헤더 자동 전파부분 구현CpfHeaderPropagator, CpfWebClientConfig, CpfRestClientInterceptor, 관련 단위 테스트하위 mock API 수신 기준 E2E 검증은 다음 보강 후보입니다.
PFW온라인 거래 로그와 헤더 detail 저장부분 구현LoggingAspect, CpfHeaderSnapshot, CpfHeaderMasker, AdmLogQueryServiceADM runtime에서 실제 거래 로그 샘플 조회는 미검증입니다.
PFW온라인 거래 메타와 로그 정책부분 구현CpfTransactionMetaScanner, LogPolicyResolver, pfw_transaction_meta, pfw_log_policyruntime smoke와 다중 인스턴스 broker 전파는 미검증입니다.
PFW/BATBatch 공통 API와 CPF Batch 메타부분 구현CpfBatchLauncher, CpfBatchOperationRepository, CpfBatchHeartbeatService, pfw_batch_*, BATCH_*ghost 자동 감지 scheduler와 장애 시나리오 테스트가 남았습니다.
ADM운영자, 권한, 회원, 캐시, 메시지, 코드, 설정 API/UI부분 구현adm/src/main/java/cpf/adm/opr/controller, adm/src/main/resources/static/adm/adm.js브라우저 실제 클릭 자동화와 UX 상태 검증은 미검증입니다.
ADM배치 관제부분 구현AdmBatchController, AdmBatchOperationService, ADM 정적 UI markerruntime API 호출, 브라우저 클릭, 관계구조도 UX는 미검증 또는 다음 보강입니다.
ADM거래/오류/감사 관제부분 구현AdmLogController, AdmObservabilityController, AdmAuditLogController, AdmLogPolicyAuditController거래 상세 탭, JSON/고정길이 전문 포맷팅의 브라우저 검증은 미검증입니다.
CMN프로젝트 공통 채번, 알림, 업무 로그부분 구현CmnSequenceService, CmnNotificationLogService, CmnBusinessLogService프로젝트별 정책과 실DB 동시성 검증은 환경별로 재확인해야 합니다.
XYZ EDU온라인 조회 샘플부분 구현XyzQueryEducationController, XyzQueryEducationService, XyzQueryEducationRepository, XyzQueryEducationMapper.xml, XyzQueryEducationMapperSliceTestXML 직접 삽입 금지, whitelist 정렬, 직접 JOIN 금지는 기본 test로 확인합니다. DB fixture 실행은 안전한 테스트 DB 환경변수 기반 실실행이 필요합니다.
SQLcmn_edu_query_item 설치 SQL 정합성부분 구현20_cmn_schema.sql, 70_test_data.sql, 99_smoke_check.sql, V14__edu_query_sample.sql, 00_all_install.sql, 00_all_install_and_smoke.sql신규 빈 MariaDB 전체 설치는 cleanup 위험 때문에 미검증입니다.
문서README와 HTML 가이드부분 구현README, specs/index.html, 개발/관리자/운영/SQL/헤더/배치 가이드기능이 늘 때마다 문서와 EDU 샘플 연결을 계속 갱신해야 합니다.

순위항목분류이유

4EDU Mapper slice DB 실행과 문서 샘플 확장EDU 품질개발자가 그대로 참고할 수 있는 실전형 샘플을 강화합니다.

7center-cut 기본 구현체와 EDU 샘플미구현대량 파일/전문 처리 표준 샘플로 남아 있습니다.
