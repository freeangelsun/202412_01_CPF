CPF 기능 구현 매트릭스

이 문서는 기능별 구현 상태, 증거 파일, 검증 기준, 남은 리스크를 한눈에 보기 위한 검수용 문서입니다. 완료는 실제 파일과 테스트 증거가 있을 때만 사용합니다.

상태 값 기준

상태의미
완료소스, SQL, 문서, 검증 증거가 현재 저장소 기준으로 존재합니다.
부분 구현핵심 구조는 있으나 운영 시나리오, UI, 자동화, 외부 연동 일부가 남았습니다.
미구현요구 기능의 실체가 아직 없습니다.
미검증구현은 있으나 이번 작업에서 실제 검증을 실행하지 않았습니다.
실패검증을 실행했고 실패했습니다.
재확인 필요환경 의존 또는 요구 해석이 남아 추가 확인이 필요합니다.

핵심 기능 상태

영역기능상태증거남은 리스크
PFW거래 ID, 거래 헤더, 표준 응답, 예외, 거래 로그부분 구현TransactionIdGenerator, TransactionContextFilter, TransactionHeaderValidationInterceptor, LoggingAspect, TransactionLogService, LogPolicyResolver, pfw_transaction_log거래 메타와 로그 정책 런타임 적용은 보강됐고, 부하 검증과 다중 인스턴스 전파는 남아 있습니다.
온라인온라인 거래 메타부분 구현@CpfTransaction, TransactionHeaderValidationInterceptor, pfw_transaction_meta, AdmTransactionMetaController브라우저 클릭 검증과 대량 운영 UX 고도화는 남아 있습니다.
온라인온라인 거래 메타 자동 등록완료CpfTransactionMetaScanner, CpfTransactionMetaRepository, CpfTransactionMetaAutoConfiguration, scripts/smoke-transaction-meta-runtime.ps1로컬 MariaDB에 V11/V12 적용 후 ADM 기동 scan, upsert, 목록, 상세 조회를 검증했습니다. 브라우저 UX 고도화는 별도 항목입니다.
온라인온라인 transactionGlobalId완료TransactionContextFilter, TransactionContext, TransactionIdGenerator외부 시스템 간 end-to-end 추적 검증은 별도입니다.
온라인온라인 오류 로그부분 구현LoggingAspect, TransactionLogService, pfw_transaction_log.ERROR_MESSAGE, pfw_transaction_log_detail별도 pfw_error_log 테이블은 없습니다.
온라인온라인 감사 로그부분 구현adm_audit_log, ADM 운영 조치 API모든 온라인 업무 변경 API 감사 강제는 아직 범위가 넓지 않습니다.
온라인온라인 파일 로그 정책부분 구현LoggingAspect, LogPolicyResolver, LogPolicyCache, pfw_log_policy거래별 fileLogLevel과 OFF 정책은 런타임 적용됩니다. 다중 인스턴스 즉시 전파는 남아 있습니다.
온라인온라인 DB 로그 정책부분 구현TransactionLogListener, TransactionLogService, TransactionLogMapper.xml, pfw_log_policy_override, scripts/smoke-log-policy-runtime.ps1, LogPolicyCacheTest로컬 ADM runtime에서 dbLogEnabled=N 저장 중지와 해제 후 fallback 저장 재개, request/response/error stack 저장 제외, active/future override와 DB policy fallback을 검증합니다. application 기본값과 CPF 기본값 fallback은 단위 테스트로 보완합니다. 운영 부하와 다중 인스턴스 전파 검증은 다음 단계입니다.
ADM로그 정책 override부분 구현AdmLogPolicyController, AdmLogPolicyService, pfw_log_policy, pfw_log_policy_override, pfw_log_policy_audit정책 변경 후 현재 인스턴스 cache evict/refresh는 구현됐고, broker 전파는 남아 있습니다.
PFWBatch 공통 API와 CPF Batch 메타완료CpfBatchLauncher, CpfBatchOperationRepository, CpfBatchHeartbeatService, CpfBatchRuntimeListener, pfw_batch_*ghost 주기 scheduler와 broker 장애 시나리오 검증은 다음 보강 후보입니다.
배치배치 Job 로그 정책부분 구현CpfBatchRuntimeListener, LogPolicyResolver, pfw_log_policyJob 시작 시 BATCH_JOB 정책을 조회해 ExecutionContext에 남깁니다. 별도 배치 로그 저장소 적용은 다음 단계입니다.
배치배치 Step 로그 정책부분 구현CpfBatchRuntimeListener, pfw_batch_step_execution, pfw_log_policy_overrideStep 시작 시 BATCH_STEP 정책을 조회해 ExecutionContext에 남깁니다. Step별 보존 정책 자동화는 남아 있습니다.
BATstandalone 배치 worker완료BatApplication, BatBatchRepositoryConfig, BatSmokeJobConfig실제 운영 배치 Job 확장은 업무별로 필요합니다.
BATheartbeat/progress완료V10__batch_heartbeat_progress_ghost_detection.sql, CPF_BAT_HEARTBEAT_JOB, BAT runtime smoke 결과운영 장시간 Job 부하 검증은 별도입니다.
BATghost detection부분 구현CpfBatchGhostDetectionService, pfw_batch_ghost_event, ADM ghost 후보 API주기 scheduler와 장애 시나리오 자동 검증은 남아 있습니다.
ADM배치 관제부분 구현AdmBatchController, AdmBatchOperationService, ADM 정적 UI marker브라우저 클릭 검증과 운영 정책 전체 자동화는 미검증입니다.
ADM거래/오류/감사 관제부분 구현AdmLogController, AdmLogQueryService, AdmAuditLogController, AdmObservabilityController, AdmLogPolicyAuditController, AdmObservabilityService거래 목록/상세, transactionGlobalId 별칭 검색, trace ID 검색, 업무 거래 ID 검색, 실패 로그 조회, 일반 감사 API, 정책 감사 API, 통합 추적 API를 보강했습니다. 오류 조치 상태/이력과 브라우저 탭형 UX 클릭 검증은 남아 있습니다.
ADM운영자/권한/회원/캐시/메시지/코드/설정/거래 메타/로그 정책부분 구현adm/src/main/java/cpf/adm/opr/controller, adm/src/main/resources/static/adm/adm.js브라우저 실제 클릭 자동화와 세밀한 UX 검증이 남았습니다.
CMN프로젝트 공통 채번, 알림, 업무 로그완료CmnSequenceService, CmnNotificationLogService, CmnBusinessLogService프로젝트별 확장 정책은 도입 프로젝트에서 확정합니다.
MBR회원 CRUD, 인증, token hash, 이력완료MbrController, MbrAuthService, MemberMapper실제 SSO/MFA 연동은 별도입니다.
EXS외부 연계 운영 API와 로그완료ExsOperationController, ExsFlowService실제 외부기관 연동 테스트는 별도 환경이 필요합니다.
XYZ EDU교육 샘플부분 구현xyz/src/main/java/cpf/xyz/eduBAT 유형별 샘플과 center-cut 샘플 보강이 필요합니다.
