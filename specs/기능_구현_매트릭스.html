CPF 기능 구현 매트릭스

이 문서는 CPF 기능별 구현 상태와 검증 상태를 분리해서 보여주는 기준 문서입니다. 구현 파일이 있어도 실행하지 않은 runtime, 브라우저, 전체 MariaDB 설치, broker 검증은 성공으로 쓰지 않습니다.

상태값은 완료, 부분 구현, 미구현, 미검증, 실패, 재확인 필요 여섯 가지만 사용합니다. 별도 환경이 필요한 경우에는 상태값으로 쓰지 않고 설명 칸에 사유를 적습니다.

상태 기준

상태정의
완료소스, SQL, 문서, 테스트 또는 smoke 증거가 현재 저장소와 현재 세션 검증 결과 기준으로 모두 맞습니다.
부분 구현핵심 구조는 있으나 runtime, 브라우저, 실DB, 외부 연동, E2E 중 일부가 남았습니다.
미구현구현 파일이나 동작 경로가 없습니다.
미검증구현 또는 스크립트는 있으나 이번 작업에서 실행하지 않았습니다.
실패이번 작업에서 실행했으나 실패한 항목입니다.
재확인 필요로컬 DB, 외부 환경, 브랜치 차이, 설정 차이로 다른 환경에서 재실행 확인이 필요합니다.

기능 상태

영역기능상태증거남은 확인
PFW표준 헤더 사전과 필수 헤더 검증부분 구현CpfHeaderNames, CpfHeaderSpecs, CpfInboundHeaderValidator, TransactionHeaderValidationInterceptor, CpfStandardHeaderE2eTestADM runtime/OpenAPI에서 실제 요청 기준 확인은 미검증입니다.
PFW표준 헤더 Source-Level 흐름 테스트완료CpfStandardHeaderE2eTest, check-feature-evidence.ps1필수 헤더, context, trusted proxy IP, 주체 헤더 의미 분리, 민감 헤더 마스킹, outbound 허용 정책을 소스 레벨에서 검증합니다.
PFWtrusted proxy 기반 client IP 해석부분 구현CpfTrustedProxyPolicy, CpfHeaderExtractor, LoggingAspect, CpfHeaderStandardCoverageTest단위 테스트는 있으나 실제 LB/WAF 뒤 운영 트래픽 검증은 별도 환경에서 재확인해야 합니다.
PFW업무 코드 헤더 보정 제한부분 구현CpfHeaderMutator, CpfHeaderMutatorTest, 표준_헤더_가이드.html운영 예외 API와 감사 diff 흐름은 별도 기능입니다.
PFWWebClient/RestClient 하위 호출 헤더 자동 전파부분 구현CpfHeaderPropagator, CpfWebClient, CpfWebClientConfig, CpfRestClientInterceptor, CpfStandardHeaderE2eTest소스 테스트 기준 허용 헤더 전파는 확인 대상에 포함했습니다. 실제 하위 HTTP 서버 수신 runtime 검증은 미검증입니다.
PFW온라인 거래 로그와 헤더 detail 저장부분 구현LoggingAspect, CpfHeaderSnapshot, CpfHeaderMasker, AdmLogQueryServiceADM runtime에서 실제 거래 로그 샘플 조회는 미검증입니다.
PFW온라인 거래 메타와 로그 정책부분 구현CpfTransactionMetaScanner, LogPolicyResolver, pfw_transaction_meta, pfw_log_policyruntime smoke와 다중 인스턴스 broker 전파는 미검증입니다.
PFW/BATBatch 공통 API와 CPF Batch 메타부분 구현CpfBatchLauncher, CpfBatchOperationRepository, CpfBatchHeartbeatService, pfw_batch_*, BATCH_*ghost 자동 감지 scheduler와 장애 시나리오 테스트가 남았습니다.
ADM운영자, 권한, 회원, 캐시, 메시지, 코드, 설정 API/UI부분 구현adm/src/main/java/cpf/adm/opr/controller, adm/src/main/resources/static/adm/adm.js브라우저 실제 클릭 자동화와 UX 상태 검증은 미검증입니다.
ADM배치 관제부분 구현AdmBatchController, AdmBatchOperationService, ADM 정적 UI markerruntime API 호출, 브라우저 클릭, 관계구조도 UX는 미검증 또는 다음 보강입니다.
ADM거래/오류/감사 관제부분 구현AdmLogController, AdmObservabilityController, AdmAuditLogController, AdmLogPolicyAuditController거래 상세 탭, JSON/고정길이 전문 포맷팅의 브라우저 검증은 미검증입니다.
CMN프로젝트 공통 채번, 알림, 업무 로그부분 구현CmnSequenceService, CmnNotificationLogService, CmnBusinessLogService프로젝트별 정책과 실DB 동시성 검증은 환경별로 재확인해야 합니다.
XYZ EDU온라인 조회 샘플부분 구현XyzQueryEducationController, XyzQueryEducationService, XyzQueryEducationRepository, XyzQueryEducationMapper.xml, xyz/src/test/java/cpf/xyz/edu/repository/XyzQueryEducationMapperSliceTest.java, xyz/src/test/resources/sql/xyz_edu_query_fixture.sql정적 XML 안전성, fixture 범위, 로컬 MariaDB fixture 실행은 확인했습니다. 전체 MariaDB 설치와 runtime 화면 검증은 별도입니다.
SQLcmn_edu_query_item 설치 SQL 정합성부분 구현20_cmn_schema.sql, 70_test_data.sql, 99_smoke_check.sql, V14__edu_query_sample.sql, 00_all_install.sql, 00_all_install_and_smoke.sql신규 빈 MariaDB 전체 설치는 cleanup 위험 때문에 미검증입니다.
문서README와 HTML 가이드부분 구현README, specs/index.html, 개발/관리자/운영/SQL/헤더/배치 가이드기능이 늘 때마다 문서와 EDU 샘플 연결을 계속 갱신해야 합니다.

검증 상태

검증상태기준비고
정적 ADM UI marker완료scripts/smoke-adm-ui.ps1DOM marker와 API route 문자열 기준 검증입니다. 실제 브라우저 클릭 검증은 아닙니다.
ADM runtime미검증scripts/smoke-adm-runtime.ps1ADM 기동, health, 거래/배치/캐시 API 호출이 필요합니다.
OpenAPI runtime미검증scripts/smoke-openapi.ps1실제 앱 기동 후 JSON 접근과 품질 확인이 필요합니다.
ADM browser click미검증브라우저 자동화 또는 수동 검증 절차정적 marker 검사를 브라우저 클릭 성공으로 보지 않습니다.
로그 정책 runtime smoke미검증scripts/smoke-log-policy-runtime.ps1실행하지 않았다면 성공으로 기록하지 않습니다.
MariaDB 전체 신규 설치미검증00_all_install.sql, 00_all_install_and_smoke.sql안전한 빈 DB 또는 명시 승인 후에만 실행합니다.
EDU Mapper DB slice재확인 필요XyzQueryEducationMapperSliceTest, sql/xyz_edu_query_fixture.sql현재 세션에서 로컬 MariaDB fixture 실행은 수행했습니다. 다만 빌드 산출물은 저장소 소스가 아니므로 다른 PC에서는 환경변수 지정 후 재실행해야 합니다.
PFW 표준 헤더 Source-Level 흐름 테스트완료CpfStandardHeaderE2eTestinbound 검증, context 생성, trusted proxy IP, 민감 헤더 마스킹, outbound 허용 정책은 소스 테스트로 확인합니다. Runtime E2E 완료로 확대하지 않습니다.
표준 헤더 Runtime E2E미검증실제 API 호출, DB 로그 저장, ADM 조회, 하위 HTTP 서버 수신앱 기동과 테스트용 하위 서버가 필요하므로 이번 작업에서는 완료로 기록하지 않습니다.
ADM 거래 로그 연계미검증AdmLogController, AdmLogQueryService거래 로그 상세 runtime 호출과 DB 적재 결과는 아직 확인하지 않았습니다.
outbound 실제 HTTP 수신 검증미검증CpfWebClientConfig, CpfRestClientInterceptor소스 테스트는 전파 대상 map을 확인하며, 실제 하위 mock 서버 수신은 다음 runtime 테스트 대상입니다.
Redis/Kafka/MQ broker미검증실 broker와 장애 시나리오기본 local test 성공으로 대체하지 않습니다.
qualityGate완료.\gradlew.bat qualityGate --offline현재 작업 세션에서 최종 실행했고 통과했습니다.
check-html-docs완료scripts/check-html-docs.ps1HTML 구조, Markdown 잔재, 불필요 산출물, 기능 매트릭스 상태값 위반을 검사합니다.
check-feature-evidence완료scripts/check-feature-evidence.ps1정본 fixture 경로, legacy fixture 잔존, DB username 환경변수, 주요 feature evidence를 검사합니다.
check-utf8완료scripts/check-utf8.ps1 -CheckMojibake현재 스크립트 기준 UTF-8과 mojibake 검사가 통과했습니다.

다음 보강 우선순위

순위항목상태이유
1ADM runtime/OpenAPI smoke와 브라우저 클릭 자동화미검증운영자 기능은 실제 화면과 API가 같이 검증되어야 합니다.
2MariaDB 안전 테스트 DB에서 전체 설치, seed, idempotent 검증미검증설치 SQL은 신규 프로젝트 재현성의 핵심입니다.
3표준 헤더 Runtime E2E와 ADM 로그 상세 검증미검증Source-Level 흐름은 완료했지만 실제 앱 기동 후 거래 로그 detail, ADM 조회, 하위 HTTP 수신까지 확인해야 합니다.
4EDU Mapper DB fixture 재현 절차 자동화재확인 필요로컬 DB 환경변수 기반 실행을 다른 개발자 PC에서도 같은 방식으로 재현해야 합니다.
5ghost 자동 감지 scheduler와 batch 장애 시나리오부분 구현장시간 배치와 비정상 종료 대응은 운영 프레임워크의 핵심입니다.
6로그 정책 다중 인스턴스 broker 전파미검증Redis/Kafka/MQ 실 broker 기준으로 cache refresh와 동적 로그레벨을 확인해야 합니다.
7center-cut 기본 구현체와 EDU 샘플미구현대량 파일/전문 처리 표준 샘플로 남아 있습니다.
