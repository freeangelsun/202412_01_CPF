CPF 배치 개발 가이드

BAT 모듈에서 Spring Batch Job을 실행하고, PFW Batch 공통 API와 CPF 운영 메타를 연결하는 기준입니다.

PFW는 배치 공통 계약과 운영 메타를 제공합니다. BAT는 실제 worker입니다. ADM은 조회와 조치 요청을 담당하며, 업무 모듈은 업무 데이터와 업무별 adapter를 소유합니다.

BAT 구조

파일역할검토 기준
bat/src/main/java/cpf/bat/BatApplication.javaBAT standalone boot entry:bat:bootJar와 runtime smoke로 기동 가능해야 합니다.
BatBatchRepositoryConfigSpring Batch JobRepository 구성Spring Batch BATCH_*와 CPF pfw_batch_*가 같은 실행 흐름으로 연결됩니다.
BatSmokeJobConfig정상, 실패, heartbeat, center-cut smoke Job 구성CPF_BAT_SMOKE_JOB, CPF_BAT_HEARTBEAT_JOB, CPF_BAT_CENTER_CUT_JOB, CPF_BAT_FAIL_JOB을 제공합니다.
BatHealthControllerBAT health와 smoke Job 수동 실행 API/bat/api/health, /bat/api/smoke/jobs/{jobId}/run으로 검증합니다.
scripts/smoke-bat-runtime.ps1BAT runtime smokejar 기동, health, 정상 Job, heartbeat Job, center-cut Job, 실패 Job, cleanup을 확인합니다.

영역테이블용도

heartbeat와 ghost

CpfBatchRuntimeListener는 Job/Step 시작과 종료를 기록합니다. 장시간 실행 Tasklet 또는 Chunk 처리 루프에서는 CpfBatchHeartbeatService.recordStepProgress를 주기적으로 호출해 처리 건수, 진행률, 현재 Step, 마지막 heartbeat를 갱신합니다.

기능구현 파일검증
heartbeat 갱신CpfBatchHeartbeatService, BatHeartbeatSmokeTaskletscripts/smoke-bat-runtime.ps1에서 heartbeatCount= step log를 확인합니다.
ghost 후보 감지CpfBatchGhostDetectionService, CpfBatchOperationRepository.detectGhostCandidatesADM batch API 조회 또는 별도 scheduler에서 후보 감지를 호출합니다.
lockCpfBatchLockManager, pfw_batch_lock동일 Job/parameter 중복 실행을 막고 stale lock은 운영 조치로 해제합니다.

center-cut 1차 기준

center-cut은 대량 파일, 전문, 대량 업무 대상처럼 여러 item을 잘라 처리하는 표준 흐름입니다. PFW는 표준 계약과 상태를 제공하고, BAT는 기본 sample worker를 제공합니다. 업무 모듈은 업무별 target/item/result 저장소와 adapter를 구현합니다.

구분파일 또는 테이블역할
PFW 계약CenterCutTargetProvider, CenterCutHandler, CpfCenterCutService대상 조회, RUNNING 표시, 처리, 결과 반영, 자식 거래 ID 연결 흐름을 표준화합니다.
상태CpfCenterCutStatusREADY, RUNNING, SUCCESS, FAILED, SKIPPED, RETRY_REQUESTED, STOP_REQUESTED를 사용합니다.
BAT sampleBatCenterCutSampleTargetProvider, BatCenterCutSampleHandler, BatCenterCutSmokeTaskletBAT standalone에서 center-cut 흐름을 실행 가능한 smoke Job으로 제공합니다.
SQLbat_center_cut_job, bat_center_cut_parameter, bat_center_cut_item, bat_center_cut_result운영 관제용 표준 모수와 결과 이력을 제공합니다.

기본 실행 명령

.\gradlew.bat :bat:test --offline
.\gradlew.bat :bat:bootJar --offline
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/smoke-bat-runtime.ps1

BAT runtime smoke는 CPF_BAT_CENTER_CUT_JOB을 호출하고 step log에 centerCutRequested=, centerCutSuccess= 증거가 있는지 확인합니다.
