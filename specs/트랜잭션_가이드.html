CPF 트랜잭션 가이드

CPF 온라인 거래는 거래 글로벌 ID, trace/span, 업무 거래 ID, 표준 헤더, 요청/응답/오류 로그를 하나의 흐름으로 묶어 추적합니다.

거래 글로벌 ID

구성길이설명
yyyyMMddHHmmssSSS17생성 시각입니다. 로그 일자와 다르면 유입 헤더 재사용, 테스트 데이터, 시계 차이를 점검합니다.
module ID3cpf.framework.module-id 또는 application name에서 산출한 모듈 코드입니다.
WAS ID7cpf.framework.was-id 기준의 서버 식별자입니다.
sequence7 기본일자별 증가 순번입니다. 설정에 따라 4자리에서 12자리까지 허용합니다.

처리 흐름

순서구현역할
1TransactionContextFilter거래 ID를 검증하거나 생성하고 trace/span과 표준 헤더를 요청 컨텍스트에 저장합니다.
2TransactionHeaderValidationInterceptor필수 헤더와 @CpfTransaction 메타 정보를 검증합니다.
3LoggingAspect요청 시작/종료/오류, 헤더 스냅샷, 로그 정책 적용 결과를 거래 로그로 저장합니다.
4CpfHeaderPropagator하위 호출에 필요한 표준 헤더를 자동 전파합니다.
5AdmLogQueryServiceADM에서 요약, 헤더, 요청, 응답, 오류, 상세, 전문을 분리해 조회합니다.

복합 거래 segment 표준

하나의 transactionGlobalId 안에서 ACC, MBR, EXS 같은 여러 모듈이 순차 또는 중첩 호출되면 각 처리 구간은 pfw_transaction_segment에 별도 저장합니다. 전체 거래 묶음은 transactionGlobalId, 개별 구간은 transactionSegmentId, 부모 관계는 parentSegmentId로 추적합니다.

항목설명구현 기준
transactionGlobalId전체 거래 묶음 ID하위 서비스 호출 시 동일하게 유지합니다.
rootTransactionGlobalId최초 유입 거래 IDX-Root-Transaction-Id 또는 최초 거래 ID로 저장합니다.
transactionSegmentId현재 처리 구간 IDTransactionSegmentService가 생성하고 X-Transaction-Segment-Id로 전파합니다.
parentSegmentId상위 구간 ID호출 깊이와 timeline tree를 만들 때 사용합니다.
transactionRole구간 역할MAIN, SUB, SHARED, EXTERNAL, BATCH, CENTER_CUT_PARENT, CENTER_CUT_CHILD, OPERATOR_ACTION
direction구간 방향INBOUND, OUTBOUND, INTERNAL, RESPONSE
durationMs, status구간별 수행시간과 성공/실패예외 발생 시 해당 구간은 FAILED로 종료합니다.

복합 거래 교육 샘플

패턴API흐름검증
APOST /acc/edu/composite/member-then-externalACC MAIN → ACC OUTBOUND → MBR SHARED → ACC OUTBOUND → EXS EXTERNALscripts/smoke-composite-transaction-runtime.ps1
BPOST /acc/edu/composite/member-calls-externalACC MAIN → ACC OUTBOUND → MBR SUB → MBR OUTBOUND → EXS EXTERNAL소스와 단위 테스트 기준 샘플로 제공합니다.
EDU 안내GET /xyz/edu/transactions/composite-sample개발자가 볼 API, ADM 조회 API, smoke 명령을 한 응답으로 안내합니다.개발 가이드의 상황별 샘플 매트릭스와 연결합니다.
