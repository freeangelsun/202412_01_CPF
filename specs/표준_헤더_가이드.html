CPF 표준 헤더 가이드

이 문서는 온라인 거래에서 주고받는 CPF 표준 헤더의 분류, 검증, 컨텍스트 조회, 하위 호출 전파, 로그 저장, ADM 표시 기준을 정의합니다.

구현 기준 클래스는 CpfHeaderNames, CpfHeaderSpec, CpfHeaderSpecs, CpfHeaderExtractor, CpfInboundHeaderValidator, TransactionContext, CpfHeaderMutator, CpfHeaderPropagator, CpfHeaderMasker, CpfRestClientInterceptor입니다.

분류 기준

분류의미운영 기준
REQUIREDCPF 온라인 거래 처리에 필요한 필수 헤더수신 인터셉터에서 누락을 차단합니다.
RECOMMENDED관측성, 추적성, 운영 편의성을 위해 항상 보내는 것을 권장하는 헤더누락 시 차단하지 않지만 로그와 ADM에서 빈 값으로 확인합니다.
OPTIONAL업무나 채널 특성에 따라 사용하는 헤더업무 서비스가 필요할 때 사용합니다.
INTERNAL_ONLYPFW 내부에서 생성하거나 응답에만 노출하는 헤더업무 코드가 임의로 변경하지 않습니다.
FORBIDDEN_TO_LOG_RAW원문 로그 저장 금지 헤더토큰, API key, 서명, 난수 값은 원문 저장과 ADM 원문 표시를 금지합니다.

주체, 채널, 호출자 헤더

그룹헤더기준
채널X-Original-Channel-Code, X-Channel-Code, X-Channel-Detail-Code최초 채널, 현재 채널, 세부 채널을 분리합니다. 업무 코드에서 세부 채널만 제한적으로 보정할 수 있습니다.
사용자/조직X-User-Id, X-Operator-Id, X-Customer-No, X-Member-No, X-Tenant-Id, X-Organization-Code, X-Branch-Code인증/권한/감사/로그 검색에 사용합니다. 고객번호와 회원번호는 필요 시 마스킹 기준을 적용합니다.
접속 위치X-Client-IP, X-Forwarded-For, Forwarded, X-Real-IP, X-Client-Country-Code, X-Client-Region-Code, X-Client-Timezoneclient가 직접 보낸 IP는 신뢰하지 않고 trusted proxy, Gateway, WAF 기준으로 해석합니다.
호출자X-Client-App-Id, X-Client-Version, X-Caller-Service, X-Caller-Instance-Id, User-Agent앱 버전별 장애 추적, 호출 서비스별 관제, 호환성 분석에 사용합니다.
보안Authorization, X-Api-Key, X-Request-Signature, X-Request-Timestamp, X-Nonce토큰, API key, 서명, 난수는 원문 저장 금지입니다. 시각 값만 감사 기준으로 저장할 수 있습니다.

자동 처리 흐름

단계구현처리 내용
수신 추출CpfHeaderExtractorHTTP 요청에서 표준 헤더를 수집하고 TransactionHeader로 변환합니다.
필수 검증CpfInboundHeaderValidator필수 헤더 누락과 X-Transaction-Id 형식을 검사합니다.
컨텍스트 저장TransactionContext현재 거래 ID, trace/span, 업무성 헤더를 요청 컨텍스트와 MDC에 저장합니다.
업무 보정CpfHeaderMutator채널 상세, 고객/회원/조직 등 업무성 헤더만 제한적으로 보정합니다.
하위 전파CpfHeaderPropagator, CpfWebClientConfig, CpfRestClientInterceptorWebClient/RestClient 호출 시 추적 헤더와 업무 헤더를 자동 전파합니다.
로그 저장LoggingAspect, CpfHeaderMaskerinboundHeaders, resolvedHeaders, outboundHeaders, responseHeaders를 마스킹 후 detail에 저장합니다.
ADM 표시AdmLogQueryService, adm.js요약, 수신 헤더, 해석 헤더, 전파 헤더, 응답 헤더, 요청, 응답, 오류, 상세, 전문 탭으로 표시합니다.

EDU 샘플 작성 기준

EDU 온라인 조회 샘플은 단건 조회, 목록 조회, offset 페이징, keyset 페이징, 검색, 정렬, DTO 매핑, MyBatis Mapper, Service 트랜잭션 경계, Controller Swagger 예시, 하위 서비스 호출 헤더 자동 전파를 각각 분리합니다. 학습용 코드에는 라인 단위에 가까운 한글 주석을 허용하고, 운영 코드에는 기능 단위 주석만 둡니다.
