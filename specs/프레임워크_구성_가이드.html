CPF 프레임워크 구성 가이드

CPF는 프레임워크 코어, 프로젝트 공통, 독립 배치 실행기, 운영 관리자, 업무 모듈, 교육 모듈을 분리합니다. 이 경계가 흔들리면 DB 소유권, 운영 권한, 배포 단위, 교육 샘플이 뒤섞이므로 신규 기능은 반드시 아래 기준으로 배치합니다.

아키텍처 경계

영역책임DB확장 방식
PFWCPF 자체 기능과 운영 메타pfwDBJava API, Filter, Interceptor, AOP, Facade, ADM 운영 API로 제공합니다.
CMN프로젝트 업무 공통cmnDB프로젝트가 필요한 업무 채번, 업무 알림, 업무 로그, 보조 기능을 개발합니다.
BAT독립 배치 실행 workerpfwDB 중심Spring Batch JobRepository와 PFW Batch 공통 API를 사용해 실제 Job과 Step을 실행합니다.
ADM운영 콘솔admDB, 조회 대상 DB권한, 감사, 로그, 배치, 캐시, 메시지, 설정을 운영자 UX로 제공합니다. Job 실행 책임은 BAT에 둡니다.
업무 모듈실제 업무 서비스업무별 DBPFW 기능은 API로 호출하고, 업무 데이터는 자기 DB에 둡니다.
XYZ EDU교육과 샘플선택개발자가 상황별로 따라 할 수 있는 학습용 코드를 제공합니다.

BAT 독립 실행 구조

항목구현기준
모듈batsettings.gradle에 독립 Gradle 모듈로 등록합니다.
애플리케이션BatApplicationjava -jar와 --spring.profiles.active=bat-local로 단독 기동할 수 있어야 합니다.
JobRepositoryBatBatchRepositoryConfigSpring Batch 표준 BATCH_* 테이블을 PFW DB에 기록합니다.
CPF 운영 메타CpfBatchLauncher, CpfBatchOperationRepositorypfw_batch_execution, pfw_batch_step_execution에 CPF 관제 메타를 기록합니다.
검증 JobCPF_BAT_SMOKE_JOB, CPF_BAT_FAIL_JOB정상 흐름과 실패 흐름을 모두 남겨 Spring Batch ID와 CPF ID 연결을 확인합니다.
운영 API/bat/api/health, /bat/api/smoke/jobs/{jobId}/runPFW 표준 거래 헤더와 @CpfTransaction 메타를 적용합니다.
검증 스크립트scripts/smoke-bat-runtime.ps1jar 기동, health, Job 실행, ID 연결, 프로세스 cleanup을 한 번에 확인합니다.

배치 메타 역할

구분책임주요 테이블운영 기준
Spring BatchJobRepository 원천 실행 메타BATCH_*Job instance, execution, step 원천 이력을 보관합니다.
CPF BatchADM 운영 메타pfw_batch_job, pfw_batch_schedule, pfw_batch_execution, pfw_batch_worker, pfw_batch_lock, pfw_batch_ghost_event관제, 수동 실행, 재수행, 중지, worker heartbeat, ghost 조치, lock을 관리합니다.
ADM관제와 조치조회 대상은 BATCH_*, pfw_batch_*운영자가 상태를 보고 조치하지만, 실제 Job 실행 책임은 BAT에 둡니다.
