프레임워크 구성 가이드

CPF의 모듈, DB, 배포 profile, migration, 관측성 기준을 설명합니다.

모듈 경계

모듈책임
pfw거래 ID, 표준 헤더, 거래 로그, 공통 응답/예외, OpenAPI, WebClient, workflow, 보안 메타
cmn공통 캐시/MQ/유틸과 선택형 cmnDB 업무 공통 기능
adm운영자, 권한, 세션, 감사, 동적 로그레벨
acc, mbr업무 샘플 모듈
xyz교육 API와 개발 샘플

DB 소유권

DB소유 데이터
pfwDB프레임워크 메타, 거래로그, 응답코드/메시지, 설정, 캐시 이벤트, JWT/security 메타
cmnDB공통 채번, 채번 발급 로그, 공통 알림 로그, 공통 업무 로그
admDB운영자, 역할, 메뉴, 세션, 감사 로그, 동적 로그레벨 룰
accDBacc_account
mbrDBmbr_member

Profile Matrix

profile기준
local로컬 MariaDB/Redis/Kafka optional. CMN business DB는 필요 시 활성화
dev/test테스트 DB와 broker를 연결하고 seed/smoke를 자동 검증
stg운영과 같은 권한/secret 구조로 사전 검증
prodsecret 기본값 금지. 환경변수, Vault, KMS, Secret Manager 사용

Migration

Flyway 형식의 V1__cpf_baseline_install.sql을 기준으로 채택합니다. 설치 SQL과 migration SQL은 목적을 분리하고, 운영 migration은 cpf_*_migration 계정으로만 수행합니다.

Observability

항목기준
healthactuator health, readiness, liveness를 profile별 노출 정책으로 관리
traceX-Transaction-Id, trace id, span id를 로그와 응답에 연결
metrics요청 수, 오류 수, 외부 API latency, DB pool, broker 상태를 수집
로그민감정보 마스킹, 보존기간, 검색 키워드 기준을 운영/보안 가이드에 연결
