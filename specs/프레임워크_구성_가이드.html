CPF 프레임워크 구성 가이드

CPF는 프레임워크 코어, 프로젝트 공통, 배치 실행 worker, 운영 콘솔, 업무 모듈, 교육 모듈을 분리합니다. 이 경계는 DB 소유권, 배포 단위, 운영 권한, 테스트 기준을 결정하므로 모든 신규 기능은 아래 기준에 맞춰 배치합니다.

아키텍처 경계

영역책임DB확장 방식
PFWCPF 자체 기능과 운영 메타pfwDBJava API, Filter, Interceptor, AOP, Facade, ADM 운영 API로 제공합니다.
CMN프로젝트 업무 공통cmnDB프로젝트가 필요한 업무 채번, 업무 알림, 업무 로그, 보조 기능을 개발합니다.
BAT업무 배치 실행 workerpfwDB 중심, 업무 DB 선택Spring Batch JobRepository와 PFW Batch API를 사용해 실제 Job과 Step을 실행합니다.
ADM운영 콘솔admDB, 조회 대상 DB권한, 감사, 로그, 배치, 캐시, 메시지, 코드, 설정을 운영자 UX로 제공합니다. Job 실행 책임은 BAT에 둡니다.
업무 모듈실제 업무 서비스업무별 DBPFW 기능은 API로 호출하고, 업무 데이터는 자기 DB에 둡니다.
XYZ EDU교육과 샘플선택개발자가 따라 할 학습용 코드를 운영 코드와 분리해 제공합니다.

모듈 구조

모듈주요 경로기준 문서완료 판단
PFWpfw/src/main/java/cpf/pfw개발 가이드, SQL 가이드공통 기능 API와 테스트가 존재하고 업무 모듈이 직접 DB를 만지지 않습니다.
CMNcmn/src/main/java/cpf/cmn개발 가이드, EDU 샘플optional datasource 사용/미사용 케이스를 모두 설명합니다.
BATbat/src/main/java/cpf/bat배치 개발 가이드, 운영 매뉴얼standalone jar, JobRepository, CPF 운영 메타, smoke 검증이 연결됩니다.
ADMadm/src/main/java/cpf/adm관리자 가이드, 운영 매뉴얼UI 숨김뿐 아니라 서버 권한검사, 감사 사유, diff, 마스킹 기준을 만족합니다.
업무 예제bizadm, mbr, exs, acc개발 가이드, 기능 구현 매트릭스실제 DB 기반 Controller/Service/Repository/Test 흐름이 있습니다.

BAT 실행 구조

항목구현기준
모듈 등록settings.gradle의 bat 모듈배치 실행은 별도 Gradle 모듈과 standalone jar로 관리합니다.
애플리케이션BatApplicationjava -jar와 --spring.profiles.active=bat-local 방식으로 독립 실행할 수 있어야 합니다.
JobRepositoryBatBatchRepositoryConfigSpring Batch 표준 BATCH_* 테이블을 사용합니다.
CPF 운영 메타CpfBatchLauncher, CpfBatchOperationRepositorypfw_batch_execution, pfw_batch_step_execution에 관제용 메타를 남깁니다.
검증scripts/smoke-bat-runtime.ps1jar 기동, health, 정상/실패 Job, ID 연결, cleanup을 한 번에 확인합니다.

center-cut 확장 기준

center-cut은 PFW 고정 테이블로 밀어 넣지 않습니다. PFW는 표준 인터페이스, 상태, 오류, 로그, 감사, transactionGlobalId 기준을 제공하고, BAT는 기본 구현체를 확장할 수 있습니다. 업무 영역은 업무별 target/item/result 테이블과 adapter를 소유합니다.

소유자소유 대상이유
PFW공통 API, 공통 상태, 공통 오류, 공통 로그여러 업무가 공유할 수 있는 계약만 제공합니다.
BAT실행 worker, 기본 center-cut adapter, 운영 메타 연결대량 처리와 실행 제어가 batch runtime에 속합니다.
업무 모듈업무 대상, item, result, 실패 데이터업무 데이터 구조와 보존 정책은 업무마다 다릅니다.
