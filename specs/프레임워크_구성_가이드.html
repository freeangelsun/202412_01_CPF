CPF 프레임워크 구성 가이드

CPF는 pfw, cmn 공통 모듈을 업무 서비스가 의존하는 구조입니다. 업무 서비스는 독립 WAS로 실행되며 공통 거래/로그/캐시/메시지/보안 기능을 공유합니다.

1. 개요

acc / mbr / xyz / adm
  |       |       |     |
  +-------+-------+-----+--> pfw: transaction, logging, exception, webclient, workflow
  +-------+-------+-----+--> cmn: code/message/config cache, security, MQ, telegram, file exchange

저장소명과 제품 표준명은 cpf-coreflow-platform-framework입니다. Java package는 cpf.*입니다.

3. PFW

기능구현

표준 예외/응답FpsException, FpsGlobalExceptionHandler, FpsResponseCodeResolver
서비스 호출FpsWebClient, cpf.services.*.base-url
워크플로우@FpsWorkflow, @FpsWorkflowStep, FpsWorkflowContext
동적 로그레벨DynamicTransactionLogLevelService, dynamic_log_level_rule 동기화

OpenAPI 화면 자체는 보통 DB 없이 로딩됩니다. 실제 API 호출은 DB/캐시/외부연계 상태를 따릅니다.

7. 실행/검증

.\gradlew.bat :pfw:compileJava :cmn:compileJava :adm:compileJava :acc:compileJava :mbr:compileJava :xyz:compileJava --offline
.\gradlew.bat :adm:bootRun
.\gradlew.bat :xyz:bootRun
패키지/설정 잔존 확인은 rg로 legacy package/import/property 패턴을 검색합니다.

8. 확장 포인트

영역다음 단계
동적 로그레벨DB polling에서 Redis/Kafka Pub/Sub 실시간 전파로 확장
ADM 보안JWT signing key Vault/KMS 연동, 권한별 버튼/행위 제어 강화
DB 검증실제 MariaDB에서 전체 SQL 실행, FK/index/권한 검증
관측성Actuator, Micrometer, trace backend 연동
