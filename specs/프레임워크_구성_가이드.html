ADR 0001. 주제영역 분리 결정

CPF는 PFW, CMN, ADM, ACC, MBR의 DB와 테이블 소유권을 분리합니다. PFW 메타와 업무 공통 데이터를 같은 DB에 섞으면 프레임워크 배포와 업무 확장이 서로 강하게 결합되므로, CMN은 jar로 함께 배포하되 cmnDB datasource는 선택형으로 둡니다.

결정결과
pfwDB는 프레임워크 엔진, 메타, 거래로그, 시스템 응답코드, 설정, 보안 메타, 캐시 이벤트를 소유합니다.프레임워크 배포 단위와 운영 메타가 업무 DB와 섞이지 않습니다.
cmnDB는 공통 채번, 알림 로그, 공통 업무 로그 같은 업무 공통 기능을 소유합니다.공통 업무 기능이 필요한 프로젝트만 DB를 붙여 사용할 수 있습니다.
admDB는 운영자, 권한, 메뉴, 세션, 감사 로그, 동적 로그레벨을 소유합니다.운영 기능과 업무 데이터 권한을 분리해 감사와 통제를 단순화합니다.
accDB, mbrDB는 업무 샘플 DB이며 각 업무 prefix 테이블만 가집니다.신규 업무 모듈은 자기 DB 또는 명확한 공통 DB만 참조합니다.

Migration

Flyway 형식의 V1__cpf_baseline_install.sql을 기준으로 채택합니다. 설치 SQL과 migration SQL은 목적을 분리하고, 운영 migration은 cpf_*_migration 계정으로만 수행합니다.

파일기준
V1__cpf_baseline_install.sql신규 환경 전체 baseline
V4__batch_schedule_repository_notification.sql기존 환경에 Spring Batch 표준 저장소, 배치 스케줄 정책, 관계/수행대상, 운영 알림 테이블을 추가

SQL/Flyway 운영 기준

설치 SQL과 운영 migration은 SQL 가이드를 정본으로 사용합니다. 분리 SQL은 유지보수 기준이고, 00_all_install.sql, 00_all_install_and_smoke.sql는 SOURCE 호출 없이 본문 전체를 포함한 단일 실행 파일이어야 합니다.

항목기준
계정 권한migration 계정만 DDL 권한을 가지고, app 계정은 DML 권한만 가집니다.
Spring BatchBATCH_*는 Spring Batch JobRepository 표준 테이블이므로 대문자 명칭을 유지하고, CPF 운영 메타는 pfw_batch_*를 사용합니다.
캐시 이벤트pfw_cache_refresh_event는 Redis/Kafka 장애 또는 미사용 환경에서 다중 WAS 캐시 갱신을 추적하는 DB fallback입니다.
검증MariaDB 실DB 검증은 FK/index, seed idempotent, smoke SQL, 계정 권한을 분리해 기록합니다.

운영/보안 구조

항목기준
Secretprod profile에는 DB password, JWT secret, crypto key 기본값을 두지 않고 환경변수, Vault, KMS, Secret Manager로 주입합니다.
시큐어코딩parameter binding, 파일 경로 정규화, 민감정보 마스킹, CORS/CSRF profile별 명시 설정, 내부 오류 은닉을 기본으로 합니다.
ADM 운영 감사비밀번호 초기화, 잠금 해제, 권한 변경, 배치 실행/재수행/중지는 감사 사유를 필수로 받습니다.
장애 대응DB 백업, 설치 SQL 버전, Flyway migration 버전, trace id, 운영 작업 로그를 함께 확인합니다.
