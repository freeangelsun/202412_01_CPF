CPF 프레임워크 구성 가이드

이 문서는 CPF의 모듈 경계, DB 구성, profile, 배치 구조, 품질 gate, 로컬 실행 기준을 정리합니다. 프레임워크 코어와 프로젝트 공통의 경계를 흔들지 않는 것이 가장 중요한 원칙입니다.

PFW는 프레임워크가 자체 제공하는 기능과 공통 API를 담당합니다. CMN은 CPF를 적용하는 프로젝트에서 개발자가 확장하는 프로젝트 공통 영역입니다.

DB 구성과 책임

DB책임대표 테이블주의
pfwDB프레임워크 코어와 운영 메타pfw_transaction_log, pfw_message, pfw_response_code, pfw_batch_job, BATCH_*업무 모듈은 PFW API를 통해 사용합니다.
cmnDB프로젝트 업무 공통cmn_sequence, cmn_notification_log, cmn_business_logoptional datasource입니다. 미사용 프로젝트도 기동 가능해야 합니다.
admDB운영자와 권한adm_operator, adm_role, adm_audit_log운영 행위는 감사 로그를 남깁니다.
bizadmDB업무 관리자 기본 구현bizadm_admin_user, bizadm_customer, bizadm_masking_auditsample 저장소를 사용하지 않습니다.
mbrDB회원 기본 구현mbr_member, mbr_login_history, mbr_refresh_tokentoken 원문은 저장하지 않습니다.
exsDB대외연계 기본 구현exs_token_store, exs_transaction_log, exs_retry_log대외 token 원문은 저장하거나 반환하지 않습니다.

Batch 구조

계층역할테이블운영 화면
Spring BatchJobRepository, JobExplorer, JobOperator, JobLauncher 기반 실행 메타BATCH_JOB_INSTANCE, BATCH_JOB_EXECUTION, BATCH_STEP_EXECUTIONADM 배치 상세에서 원천 실행 메타를 조회합니다.
CPF Batch운영 job 등록, 스케줄, lock, 수동 실행, 재수행, 중지, 이력pfw_batch_job, pfw_batch_schedule, pfw_batch_execution, pfw_batch_lockAdmBatchController와 ADM 화면이 사용합니다.
EDU BatchTasklet, Chunk, retry, restart, lock 샘플필요 시 PFW Batch 사용XyzBatchEducationController에서 개발자가 흐름을 학습합니다.
